Buyer and Supplier Workflow

Invoices are exchanged between buyers and suppliers following the process below:

1.The supplier creates and submits an invoice.

2.The invoice is processed based on the buyer’s configuration.

3.The buyer approves or disputes the invoice.

§Based on the buyer’s configuration, the buyer may be required to code verify the invoice.

§Based on the buyer’s configuration, the processing may be in the OpenInvoice Platform or the buyer’s ERP.

§If an invoice is disputed, the invoice is sent back to the supplier who can cancel it or modify and resubmit it.

4.Once the invoice is approved, it is at the end of workflow in OpenInvoice and is ready to be paid.

§If both the buyer and supplier are configured to use the Payment module, payment processing may be done through OpenInvoice.

§If the buyer determines there is an issue with the invoice after the invoice is approved, the buyer is able to return the invoice to the workflow for issue resolution using the Unapprove feature.

§Based on the buyer’s configuration, the coding on the invoice may be amended.

Performing Batch Invoice Workflow Actions

Depending on your search criteria or the counter you select in the Invoices portlet, some workflow actions may be performed in batch. If performing an action on a batch of invoices, consider that some invoices on the list page cannot be processed if you do not have the appropriate permissions or the invoice is not in the appropriate workflow status. These invoices are not differentiated on the list page and you will still be able to select them as part of the batch action. However, once you click the action button, an error message will appear stating that one or more of the invoices cannot be processed.

Batch invoice workflow actions can be performed on the following list pages which are accessed from the Invoices portlet:

·Ready to Code

·My Ready to Code

·My Ready to Review

·Ready to Approve

·My Ready to Approve

Batch invoice workflow actions can also be performed directly in the My Ready to Approve portlet.

Note:In order to batch approve an invoice you must be assigned the Invoice Batch Approve permission.

Invoice Status

The Invoice status is assigned based on where the invoice is within the workflow.                  

Status

Description

On-Hold

Invoice cannot move through workflow until an issue has been resolved. When viewing the invoice, the status field will display On-Hold: (Pending Supplier Setup) for invoices posted from suppliers who are not completely configured or On-Hold: (Pending Duplicate Resolution) for invoices that the system has flagged as duplicate.

Submitted

Invoice has been submitted by the supplier and is not yet approved. This includes invoices that have been worked on, but the entire workflow has not been completed. For example, in the Code then Approve workflow, invoices with a Submitted status have been code verified. In the Approve then Code workflow, invoices with a Submitted status have been financially approved.

Disputed

Invoice has been disputed by the buyer and is currently waiting for supplier action. The invoice cannot be deleted but must be either modified and resubmitted or cancelled.

Re-Submitted

Invoice has been disputed by the buyer, modified by the supplier and resubmitted. This includes invoices that have been worked on, but the entire workflow has not been completed. For example, in the Code then Approve workflow, invoices with a Re-Submitted status have been code verified. In the Approve then Code workflow, invoices with a Re-Submitted status have been financially approved.

Cancelled

Disputed invoice that is subsequently cancelled by the supplier or the system if your company has configured this option. Once cancelled, the invoice has reached end-of-life and cannot be modified.

Approved

Invoice has been reviewed and approved for payment; however, it can be modified through the amend or unapprove process. An invoice with Approved status may or may not have been paid; however, the invoice cannot be unapproved if it is paid. Invoices can be flagged as paid, and the action will show up in the Invoice Journal, but there is no separate status for a paid invoice.

Things to Consider when Correcting an Image Invoice

·The Correct function allows a user to change the header information on an Image Post or Image OBO invoice that has been submitted but not yet approved. This includes invoices that are on-hold pending supplier setup. You can change the invoice number, invoice date, service date, currency, invoice amount and description, and add or remove ticket numbers. If the company preference Enable Supplier Correction is enabled, users can also modify the supplier and/or supplier site for an image invoice.

·The Invoice Journal will show the Corrected action along with any comments you entered. If the supplier name or supplier site were modified, the new and previous values will be listed.

·If the Invoice Amount field is changed, the remit amount will be automatically changed to match. If the invoice was itemized, it will have to be reitemized.

·To modify the buyer site and department, approver, or routing code on an Image OBO invoice, the invoice must be disputed.

How to Short Pay an Image Invoice

Permission:Invoice Short Pay

Prerequisite:The invoice is in Submitted or Resubmitted status.

The invoice submission type is Image OBO or Image Post.

Steps:

1.Do one of the following:

§Select Invoice > Dashboard and click one of the dashboard counters.

§Perform a search by invoice number using the Global Search field in the main navigation bar.

§Perform a search with multiple criteria by going to Invoice > Invoice Search.

2.Click Short Pay.

3.Enter the new remit amount.

Note:The value must be greater than 0.00 and less than or equal to the original amount of the invoice.

4.Enter a comment.

5.Click Submit.

Things to Consider when Short Paying an Image Invoice

·The Short Pay function allows buyers to reduce the remit amount on Image OBO or Image Post invoices. Image Scanned invoices submitted by the supplier cannot be short paid.

·Once the new remit amount is submitted, a short pay line and the reduced total appears on the Summary tab.

·If the invoice is itemized after being short paid, the short pay line is not available in the Itemization Wizard. The Itemization Wizard always shows the original amount.

·The Short Pay amount can be adjusted again if necessary. The existing short pay line is updated and a new coding record is added reflecting the change in value. If the original invoice amount is entered in the Short Pay window, the short pay line and all coding records are deleted. There will be a Short Paid entry in the Invoice Journal for every short pay action performed on the invoice.

·The short pay line must be coded. If the invoice is short paid after it has been coded, the original coding amounts are kept. The short pay line is negative; when coded, the short paid amount of the invoice will still equal the total allocations. For example, an invoice for $100 is coded: $40 to general ledger code 1000.100, $60 to 2000.200, then short paid to $90. The coding will be $40 to 1000.100, $60 to 2000.200 and ($10) to the Short Pay coding record which may be given its own code or used to offset one of the other coding records. The total allocation will be the short paid amount of $90 ($40 + $60 – $10).

·If the lines on the invoice are allocated to different codes and the short pay amount should be split between them to reduce the allocations proportionately, the short pay line must be split coded.

How to Remove an Invoice Submitted in Error from Workflow

Invoices submitted in error by a supplier can be removed from workflow by either deleting the invoice or cancelling the invoice. The method used will depend on how the invoice was created and whether you want the invoice to be included in search and reporting results. Cancelled invoices remain available in the application and are included in search results. Deleted invoices are removed from the application and are no longer available.

Invoices created and submitted by the supplier are removed from workflow by disputing the invoice to the supplier and having the supplier cancel the invoice.

Invoices created and submitted by the buyer using the Image OBO process are removed from workflow by disputing the invoice to the indexer or an internal dispute resolution department and having the indexer or a member of the dispute resolution department cancel the invoice. Alternatively, a user with the Invoice Delete (IM) permission can delete the invoice.

Invoices created and submitted by the buyer using the Image Post process are removed from workflow by having a user with the Invoice Delete (IM) permission delete the invoice.

Invoices that have been saved or modified in any way must be disputed back to the supplier for an attachment to be added.

Suppliers will receive a confirmation screen with a list of all attachments when submitting an invoice or a warning that no attachments have been included.

How to View a Field Ticket Attachment From an Invoice

Steps:

3.In the Tickets section, click the link under Attachments.

 

The attachment displays in a new window.

Note:Depending on the format of the attachment, you may have to save the file to your machine and open the downloaded file to view the attachment.

Invoice Contacts

This section describes how to manage invoice contacts.

How to Add Contacts

Permission:Invoice Modify

Prerequisite:Your company currently owns the invoice.

The invoice is in Submitted or Re-Submitted status.

Steps:

1.Do one of the following:

§Select Invoice > Dashboard and click one of the dashboard counters.

§Perform a search by Invoice Number using the Global Search field in the main navigation bar.

§Perform a search with multiple criteria by going to Invoice > Invoice Search.

2.Click the Invoice Number.

3.Above the invoice tabs, click Contact to display the contact list page.

4.Click Add Contact.

5.Do one of the following:

§Select Unregistered if your contact is not from one of your OpenInvoice trading partners.

 

§Select Registered Supplier if your contact is from one of your OpenInvoice trading partners.

 

6.Click Next.

7.Enter or select your contact’s information.

Note:Contacts are visible to both buyer and supplier companies. Contact Type, Contact Name, and Phone Number appear on the invoice. All contact details can be viewed from the contact list page.

8.Click Submit.
