What’s New

The following is a summary of key features included in OpenInvoice 2018.09.14. For a complete list of all items included in the release, refer to the release notes that are available in the Resource Center.

Release Notes

The release notes are located in the Resource Center under the Release Information, Version 2018.09.14 section.

New Features and Enhancements

This section contains a brief description of the new features added to this version of OpenInvoice.

·Purchase Orders 

§We enhanced the PIDX purchase orders to support the ship to address at the PO Line level sent to B2B suppliers.

§We enhanced the interaction when creating an invoice from a purchase order or from a field ticket that references a purchase order so that the available buyer sites matches the sites associated with the purchase order.

§There is a new interface that allows suppliers to view, accept, reject, and print purchase orders sent by their customers. These purchase orders can also be converted directly into a field ticket, copying the price book, product/service, pricing, and coding information automatically. This functionality is only available to customers who would like to be early adopters and provide feedback on the interface. Please use the OpenInvoice Feedback page to request access.

·Price books

§Suppliers can now submit price book changes directly within OpenInvoice. This new feature enables buyers and suppliers to work together more efficiently by managing price books in OpenInvoice. For buyers, this new feature will significantly reduce the effort of manual pricing updates. For suppliers, it provides increased visibility and control of pricing across all their customers. Collaboration on price books can also improve turnaround times and ultimately support increased compliance as updates can be initiated and applied quickly.

§Buyers can control when collaboration is available on the Summary section of their price books. Suppliers will be able to view price books, but if collaboration is turned off they will not be able to submit any changes.

§Suppliers can designate user permissions to modify price books, and these users will be able to submit pricing when collaboration is enabled by the buyer.

§Suppliers with the Price Book Modify permission will now see a Price Book option in their main menu. They have the option to search for a price book or they can simply reference their new dashboard widget. This new feature will provide suppliers valuable visibility into any price book they have with their buyer. User simply need the Price Book View permission to see the price books on their dashboard or to search for them from the Price Book menu option.

·Field Tickets

§We added the ability to search for and create stacks based on a purchase order. Users may also search for tickets that do not reference a purchase order.

§We enhanced the water hauling ticket type to support a unit of measure for the volume field. Current units supported are barrels (bbl), US gallons (US gal), cubic meters (m3), and litres (L). All previous volumes will remain as barrels (bbl).

§New OpenTicket supplier user interface includes: wizard-style ticket creation, advanced search criteria, ability to create stacks by multiple criteria, list view for tickets, export tickets in a summary or detailed report, view your customer’s custom labels, filter items by the price book selected, automatically populate line item details from a price book, and ability to save field tickets without submitting them to your customer. The ticket automatically saves changes when you first create it, edit the header, add a new line item, or edit an existing item. You can submit the field ticket to your customer when you have everything completed.
