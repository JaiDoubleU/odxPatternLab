Removing User Access

There are three ways of removing a user’s access to OpenInvoice:

·Deactivate a user - if they will be temporarily away from the company (short-term leave).

·Suspend a user - if they no longer need to sign in, but you want them to continue to be referenced on documents (as a requisitioner on an invoice).

·Delete a user - if they have left your company or have changed roles and will never need to access OpenInvoice in the future.

If you have removed access from a user who owns invoices in workflow, the invoices will become stalled unless someone with the Invoice Admin permission forwards the invoices to other users. A user with the Membership Admin permission may delegate the invoices to another user.

Implications of Removing Access

Deactivating a User

·Deactivated users (Inactive status) cannot be referenced on documents.

·An inactive user’s password and security questions are maintained in case the user is reactivated. If a user with a Current or Expired status is deactivated, reactivation will allow the user to immediately regain platform access. If a user with a Pending or Suspended status is deactivated, they will need to follow the registration process to be reactivated.

Note:If a user’s password expires while they are inactive, they will have 180 days to sign in and change their password before they are suspended.

Note:See the Deactivating a User section for more information.

Suspending a User

·Suspended users can be referenced on documents (as a requisitioner on an invoice).

·A suspended user’s password and security questions are no longer valid and cannot be reused. The user will need to follow the registration process to regain access to the platform.

·If a user is suspended while they are in Inactive status, their status will remain Inactive. After they are reactivated, their status will change to Suspended.

Note:See the Suspending a User section for more information.

Deleting a User

·Deleted users cannot be reactivated. If a deleted user needs access to the OpenInvoice Platform, a new username and account must be created.

·Deleted users cannot be referenced on documents, but existing references will be maintained (as a requisitioner on an invoice).

·Deleted users do not appear in the user report.

·Department notification users cannot be deleted. The user must be unassigned as the department notification user before their profile can be deleted. An error message displays if you attempt to delete a notification user.

Note:See the Deleting a User section for more information.

How to Deactivate a User

Note:If you receive a message indicating you cannot deactivate a notification user, see the Modifying a Department’s Notification User section for more information.

Use this procedure to deactivate a user.

Permission:Membership Modify

Steps:

  1. Select Membership > Users.

    2. Note:By default, all users are listed. Apply a filter to narrow the list. If only part of the criteria is known, an asterisk (*) must be used to denote the unknown part of the criteria.

  2. Click either the Last Name or First Name of the user.

  3. Click the Profile tab.

  4. Deselect the Active checkbox.

    5. Note:To reactivate the user, select this checkbox.

     

  5. Click OK to confirm.

  6. Click Submit.

How to Suspend User Access

Use these procedures to suspend access for a single user or a group of users.

To suspend a single user:

Permission:Membership Modify

Steps:

  1. Select Membership > Users.

    2. Note:By default, all users are listed. Apply a filter to narrow the list. If only part of the criteria is known, an asterisk (*) must be used to denote the unknown part of the criteria.

  2. Click either the last name or first name.

  3. Click the Summary tab.

    4.  

  4. Click Suspend Account.

  5. Click OK to confirm.

To suspend multiple users:

Permission:Membership Modify

Steps:

  1. Select Membership > Users.

    2. Note:By default, all users are listed. Apply a filter to narrow the list. If only part of the criteria is known, an asterisk (*) must be used to denote the unknown part of the criteria.

  2. Select the checkboxes next to the users’ names.

    3.  

  3. Click Suspend Users.

  4. Click Yes to confirm.
