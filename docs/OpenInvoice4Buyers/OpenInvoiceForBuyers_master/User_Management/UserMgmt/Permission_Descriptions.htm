Permission Descriptions

Permissions have different scopes that define where in the corporate structure a permission has effect. Permissions can apply to a particular document (a price book), a department, a site (all departments in the site), or to the entire corporation (all sites and all departments).

            

Price Book Permissions

To access a price book, the user must be a member of a role (with price book permissions) assigned to the price book.

Early Pay Permissions

Early Pay View

Company

View Early Pay program details.

Early Pay Modify

Company

View, create, modify, activate, deactivate, and delete Early Pay programs.

Corporate Permissions

Corporate View

Company

View sites, AFEs, cost centers, suppliers, reference data, workflow configuration, COA setup, company codes, and company preferences.

Ability to export smart coding rules.

Corporate Modify

Company

Inherits Corporate View access, plus:

Add, modify, and delete AFEs, cost centers, and reference data.

Edit COA setup.

Corporate Admin

Company

Inherits Corporate View access, plus:

Add, modify, and delete sites.

Edit workflow configuration including auto approvals, invoice data fields, corporate column confirmation, invoice recipient rules, invoice workflow rules, and smart coding.

Import smart coding rules.

Edit company preferences.

Corporate Integration

Company

Pause, resume, and manually run services.

Subscribe to integration messages (service failures / completions, system errors)

Configure company codes, integration notification users, and default e-Site and e-Department.

Corporate Partnership

Company

Inherits Corporate View access, plus:

Enable suppliers and configure supplier settings.

Add and modify unregistered suppliers.

Merge supplier records.

Assign Early Pay programs to supplier sites.

Manage supplier on-ramping.

Subscribe to notifications when a registered supplier's profile information changes.

Message suppliers.

If the user also has the Payment Admin permission, they can configure the payment schedule for all suppliers, enter bank details for unregistered suppliers, and validate the supplier for payments.

Invoice Permissions

Invoice View

Site

View, print, and comment on invoices.

Send messages.

Create custom and system reports.

Forward an invoice in possession.

Invoice Modify

Department

Inherits Invoice View access, plus:

Forward or dispute invoices.

Modify buyer-controlled invoice content (coding).

Add attachments to invoices.

Delete attachments added by users within your company.

Itemize Image Scan invoices.

Add contacts.

Invoice Admin

Company

Inherits Invoice View access, plus:

Batch forward and dispute invoices owned by another user.

Create and manage public split coding templates.

Delete public split coding templates if the creator is inactive.

Edit restricted invoice data fields entered by another user and not marked as Editable.

Route an eligible invoice currently in the OpenInvoice Platform to an external workflow.

Unlock an invoice previously routed to an external workflow to make it available for processing in the OpenInvoice Platform.

View and forward unowned invoices.

Invoice Code

Department

Inherits Invoice Modify access, plus:

Verify the coding on invoices pending code verification.

Invoice Approve

Department

Inherits Invoice View access, plus:

Dispute, forward, or comment on invoices pending financial approval.

Approve invoices pending financial approval if the user is granted an authorization level.

Invoice Batch Approve

Company

Approve multiple invoices in a single action directly from the invoice list page.

This permission does not grant access to invoices. The user must be assigned the Invoice Approve permission in one or more departments and an authorization level.

Invoice Unapprove

Site

Inherits Invoice Modify access, plus:

Unapprove a previously approved invoice and return it to the workflow.

Invoice Add Data

Site

Inherits Invoice Modify access, plus:

Add and edit restricted invoice data fields on an invoice.

Invoice Amend

Site

Inherits Invoice Modify access, plus:

Amend coding on approved invoices.

Invoice Delete (IM)

(image invoices only)

Site

Delete an Image Post invoice.

Invoice Modify (IM)

(image invoices only)

Department

Search for image files.

Modify invoice number, invoice date, service dates, currency, invoice amount, and description.

Use the Correct function to add or remove ticket numbers for invoices with either an Image OBO or Image Post submission type.

If the company preference Enable Supplier Correction is active, users can also modify the supplier and/or supplier site on an image invoice.

Create and modify Image OBO invoices.

Itemize Image OBO and Image Post invoices.

Note: This permission is considered to be Admin-level.

Invoice Short Pay

(image invoices only)

Site

Inherits Invoice Modify access, plus:

Short pay an image invoice.

Invoice Hold Adjudication

Company

Inherits Invoice View access, plus:

Release, correct, or delete duplicate invoices.

Dispute and forward duplicate invoices.

Transaction Upload

Company

Inherits Invoice View access, plus:

Upload invoices (financial transactions) in an approved or paid status.

Membership Permissions

Membership View

Company

View departments, users, roles, user licenses, and authorization levels.

Membership Modify

Company

Inherits Membership View access, plus:

Unlock users.

Reset user accounts.

Sign out users.

Add, modify, and delete users.

Add, modify, and delete non-admin roles.

Assign users to non-admin roles.

Edit non-Admin permissions in roles.

Assign licenses to users.

Receive notifications when a user’s email address has changed.

Receive notification when another administrator has turned off user email change notifications.

Run user reports.

Membership Admin

Company

Inherits Membership View access, plus:

Add, modify, and delete admin roles.

Add, modify, and delete departments.

Assign users to admin roles.

Edit admin permissions in roles.

Receive notification emails when a user locks themselves out of their account.

Receive notifications when a user’s email address has changed.

Receive notification when another administrator has turned off user email change notifications.

Run user reports.

Membership Manage Auth

Company

Inherits Membership View access, plus:

Add, modify, and delete authorization levels.

Assign authorization levels to other users.

Assign the Membership Manage Auth permissions to other users.

Receive notifications when a user’s email address has changed.

Receive notification when another administrator has turned off user email change notifications.

This permission cannot be combined with any other permission in a role.

Note: To assign authorization levels to yourself, you must also have the Self Manage attribute enabled on your profile. Refer to View and Modify Your Profile.

Payment Permissions

Payment View

Company

View payment information and configuration in the Payment module.

Receive notifications on impounds and unregistered supplier master data failures.

View Payment reports.

Payment Modify

Company

Inherits Payment View access, plus:

Modify payments in the Payment module.

Manually generate and schedule payments for approved invoices when the payment is not generated by the system.

Payment Admin

Company

Inherits Payment View access, plus:

Create and delete payment accounts.

Modify the payment account nickname and name that appears on checks.

Enter payment account contact information.

Assign sites to payment accounts.

Assign notification users to payment accounts.

Modify supplier-specific payment terms.

If the user also has the Corporate Partnership permission, they can configure the payment schedule for all suppliers, enter bank details for unregistered suppliers, and validate the supplier for payments.

Purchase Order Permissions

PO View

Site

View and print purchase orders.

PO Modify

Site

Inherits PO View access, plus:

Cancel and reopen purchase orders.

Field Ticket Permissions

Field Ticket Confirm

Department

Provides the ability to “confirm” a field ticket. Users with this permission can review and confirm a field ticket prior to final approval.

Field Ticket Bulk Approve

Department

Controls access to Bulk Approval for Field Tickets. Users with this permission can bulk approve field tickets.

Field Ticket View

Company

Create stacks and view qualifying tickets; search and view tickets by ticket number.

Field Ticket Modify

Company

Code, edit, approve and dispute tickets.

Field Ticket Modify (Restricted)

Department

Controls access to Field Tickets. Users with this permission will only see tickets assigned to them or assigned to the department where the user has restricted view permissions (using the Ticket Owner field).
