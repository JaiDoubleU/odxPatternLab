Departments

Your company consists of one or more sites. Each site consists of one or more departments. Each department consists of one or more roles. Each role has a unique set of permissions.

Implications of Editing or Deleting Departments

OpenInvoice does not check if there are tickets against the department before deleting a department.

OpenInvoice does not automatically inform suppliers when a Remit To department has been modified or deleted. The buyer must inform suppliers about any changes to the Remit To department so they can update their records and any affected integration logic.

Users affected by a modified or deleted department will retain their current permissions until they sign out. Any altered assigned permissions will take effect the next time they sign in.

Department Details

·Department Name: must be unique within the selected site, but other sites may have departments with the same department name.

Note:The department name must be unique within each site, but the department name can exist in another site (multiple sites can have an Accounts Payable department, but a single site cannot have more than one Accounts Payable department).

·Department Code: must be unique among all sites and departments. The same code cannot be used for both a site and a department. The department code is used for routing Image Post invoices.

·Site: the company location where the department exists.

·Description: additional department information that is only displayed to users within your company on the Site list page and Departments and Roles list page.

·Active: indicates if the department is active.

·Remit Invoices To: select this option to make the department available to suppliers when submitting invoices.

·OI Receipt Department: This preference is obsolete and no longer supported. It is scheduled to be removed from the platform in a future release.

·Notification: the notification user who receives an email when certain activities occur on documents in a department. The notification user does not need to be assigned any roles in the department.

Notification Users

·Notification users receive notifications about stalled invoices (based on last action), and they inherit invoice administrator capabilities (forward, dispute, etc.) if the invoice is stalled.

·The same notification user can be used for multiple departments and multiple sites.

How to Add a New Department

Use this procedure to add a new department.

Permission:Membership Admin

Steps:

  1. Select Membership > Departments & Roles.

  2. Click Add Department.

    3.  

  3. Enter a department name, select a site, and select a notification user. All other fields are optional.

    4.  

    Note:To search for a notification user, enter all or part of the user’s name and click the search icon.

  4. Click Submit.

How to Edit a Department

Use this procedure to edit an existing department.

Permission:Membership Admin

Steps:

  1. Select Membership > Departments & Roles.

  2. Click the department name.

    3.  

  3. Click the Settings tab.

  4. Click the department name.

    5.  

  5. Update the information.

    6. Note:To search for a notification user, enter all or part of the user’s name and click the Search icon.

  6. Click Submit.

How to Delete a Department

Use this procedure to delete an existing department.

Permission:Membership Admin

Steps:

  1. Select Membership > Departments & Roles.

  2. Ensure all documents assigned to the department have been processed or forwarded elsewhere.

  3. Ensure all roles associated with the department have been removed from any assigned price books.

  4. Click the delete icon.

    5.  

  5. Click OK to confirm.

Note:All roles in the department are deleted.

How to Deactivate a Department

Use this procedure to deactivate an existing department.

Permission:Membership Admin

Steps:

  1. Select Membership > Departments & Roles.

  2. Click the department name.

  3. Click the Settings tab.

  4. Click the department name.

  5. Deselect the Active checkbox.

    6.  

  6. Click Submit.

  7. Click OK to confirm.
