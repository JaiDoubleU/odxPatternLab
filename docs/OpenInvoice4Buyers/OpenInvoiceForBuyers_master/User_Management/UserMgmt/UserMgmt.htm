User Management

All companies transacting in the OpenInvoice Platform are structured into sites, departments, and roles. This structure helps control user access to various functional areas of the platform. Much like how employees perform office tasks based on their roles and responsibilities, OpenInvoice users are restricted to only those areas in the platform that pertain to their job functions. This includes limited functionality around user management.

Preventing Account Harvesting

Account harvesting refers to the illegal practice of obtaining usernames and/or passwords with the intention of gaining unauthorized access to a software system. The OpenInvoice Platform has been designed with a number of security features to prevent unauthorized access to the platform:

·When a hacker attempts to sign in and is subsequently locked out, the platform does not notify them if the username or password are incorrect. The account must be unlocked by a user with the Membership Modify permissions, who will verify that the sign in attempts were performed by the genuine user.

·Usernames are not included in the registration or reset emails in case the email is intercepted by an unauthorized party.

Displaying Internal Support Information

To set up your company’s internal support contact information you must designate a current user or create a user profile with contact information that will be displayed when users cannot access their account. In the designated user’s profile, enter DO_CONTACT_ADMIN into the Employee # field. This code prompts the system to display the designated user’s phone number and email address if a user is having problems signing in.
