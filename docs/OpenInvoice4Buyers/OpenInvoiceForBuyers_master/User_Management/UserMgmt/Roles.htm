Roles

Roles are set up from within departments. Departments separate functional areas of the company, and roles define job duties within the department. Permissions are assigned to roles according to job function. Users are assigned to roles that allow them to perform the tasks related to their job area. A set of permissions may include varying levels of access including view-only, permission to modify, and/or permission to configure.

·View permissions allow users to view documents, comment, add attachments and move the document through workflow, but they may not change the content of documents.

·Modify permissions allow users to change the content of documents.

·Admin permissions allow users to perform special functions.

Note:The Invoice Modify (IM) and Invoice Hold Adjudication permissions are considered Admin level permissions.

Considerations When Configuring Roles

The Member Role

When the department is added, an empty role called Member is automatically created. All users in a department will be assigned to the Member role, along with any other roles that are applicable to their job function. Because every user must be assigned to the department’s Member role, the Member role should be configured with only the permissions that apply to all members of the department. All permissions can be removed from the Member role if there are no common permissions that apply to all members of the department.

Note:The Member role cannot be renamed, deleted, or made inactive.

Segregation of Duties Between Membership Permissions

Users with the Membership Modify permissions can manage user profiles, create roles, add or remove non-Admin permissions to roles, and assign users to roles that do not have Admin permissions.

Users with the Membership Admin permissions can add or remove Admin permissions on roles and assign users to any role including roles with Admin permissions.

Users with the Membership Manage Auth permission can create, edit, modify, or assign authorization levels. To assign an authorization level to yourself, you must have the Self Manage attribute enabled along with the Membership Manage Auth permission.

To assign yourself to a role or modify the permissions of a role in which you are a members, you must have the Self Manage attribute enabled along with the Membership Modify and/or Membership Admin permissions. Refer to View and Modify Your Profile.

Implications of Editing or Deleting Roles

·If a role is marked Inactive, it will be displayed on the Departments and Roles list page as (Inactive). Existing users and permissions are still assigned to the role; however, the permissions will not apply to the users. An inactive role can still be edited.

·Users affected by changes to role assignments will retain their current permissions until they sign out. Any altered access to the platform will take effect the next time they sign in. Users can be forcibly signed out, if necessary.

How to Add a Role

Use this procedure to add a new role to your department.

Permission:Membership Modify and/or Membership Admin

Steps:

  1. Select Membership > Departments & Roles.

  2. Click the Department Name.

  3. Click the Settings tab.

    4.  

  4. Click Add Role.

    5.  

  5. Enter a Role Name. The Description is optional.

  6. To add users to the new role, select one or more Non-Member users and click Add >.

    7. Note:Only users who are assigned the department’s Member role will be available to add to the new role.

  7. Click Submit.

Next Steps: See the Modify the Permissions Assigned to the Role to set the permissions for the new role.

How to Edit an Existing Role

Use this procedure to add or delete members for an existing role.

Permission:Membership Modify and/or Membership Admin

Steps:

  1. Select Membership > Departments & Roles.

  2. Click the department name.

  3. Click the Settings tab.

  4. Click the Role Name in the left column.

    5.  

  5. Update the information.

  6. To add users to the role, select one or more Non-Member users and click Add >.

    7. Note:Only users who are assigned the department’s Member role will be available to add to the new role.

  7. To remove users from the role, select one or more Member users and click < Remove.

  8. Click Submit.

How to Delete a Role

Use this procedure to delete a role.

Permission:Membership Modify and/or Membership Admin

Steps:

  1. Select Membership > Departments & Roles.

  2. Click the Department Name.

  3. Click the Settings tab.

  4. Click the Delete icon.

    5.  

  5. Click OK to confirm.
