Coding Configuration

This section provides instructions on how to define the relationships between coding elements. Valid combinations of coding elements are created and applied to tickets and invoices. This section also provides instructions on how general ledger coding is applied to tickets and invoices.

The following topics are applicable to company administrators who oversee configuration of the cost allocation functionalities in OpenInvoice:

·Mapping Cost Centers to AFEs

·Mapping Cost Objects to the General Ledger

·Mapping Cost Object Types to the General Ledger

·Configuring Smart Coding

·Supplier Coding

Mapping Cost Centers to AFEs

Limit the cost centers that can be applied to a document by mapping cost centers to AFEs. If an AFE is referenced on a document, only its associated cost centers can be applied.

How to Modify the Cost Centers Mapped to an AFE

Permission:Corporate Modify

Prerequisite:AFEs and cost centers must be configured.

Steps:

1.Select Corporate > AFEs.

2.Perform a search for AFEs.

Note:If criteria is unknown, click Apply Filter to return a complete list.

3.Click the Configure Cost Center icon in the Cost Centers column.

 

4.Do one of the following:

§To add a cost center, enter a Cost Center Number and click Add >.

Note:If only part of the cost center number is known, an asterisk (*) must be used to denote the unknown part of the number. Click Search, select from a list of matching cost centers, and click Submit.

§To unmap a cost center, select a Cost Center Number in the Mapped Cost Centers column and click < Remove.

Note:To select multiple items, hold the Ctrl key and select the items. Alternatively, you can hold the Shift key to select a range of items.

 

5.Repeat step 4 for each cost center to be added or removed.

6.Click Submit.

Mapping Cost Objects to the General Ledger

Limit the general ledger codes that can be applied to a document by mapping major and minor combinations to an AFE or cost center. If an AFE or cost center is referenced on a document, only its associated general ledger codes can be applied.

How to Modify the General Ledger Accounts Mapped to Cost Objects

Permission:Corporate Modify

Prerequisite:Cost objects and the chart of accounts must be configured.

Steps:

1.Select Corporate > AFEs or Corporate > Cost Centers.

2.Perform a search for the cost object.

Note:If criteria is unknown, type an asterisk (*) into one of the fields and click Apply Filter.

3.Click the Configure Major Accounts icon in the Major/Minor column.

 

4.Do one of the following:

§To add an account mapping, select an account in the Major Accounts column, select one or more accounts in the Minor Accounts column, and click Add >.

Note:If general ledger accounts have been mapped to cost object types, only the accounts mapped to the cost object’s assigned type will be available for mapping.

Select the asterisk (*) to map the entire major account.

To select multiple items at once, hold the Ctrl key and select the items. Alternatively, hold the Shift key to select a range of items.

§To unmap an account, select an account in the Mapped column and click < Remove.

 

5.Click Submit.

Mapping Cost Object Types to the General Ledger

Limit the general codes that can be applied to a document by mapping major and minor combinations to an AFE type or cost center type. If an AFE or cost center is referenced on a document, only the general ledger codes associated with the AFE or cost center’s assigned type can be applied.

How to Modify the General Ledger Accounts Mapped to Cost Object Types

This section describes how to modify the general ledger accounts mapped to cost object types for AFE Types and Cost Center Types.

To modify the general ledger accounts mapped to cost object types for AFE Type:

Permission:Corporate Modify

Prerequisite:Cost object types and the chart of accounts must be configured.

Steps:

1.Select Corporate > COA Setup.

2.Click Coding Validation.

3.Click AFE Type under the Validation Type column.  

 

4.Click one of the AFE types under the AFE Type column.

 

5.Do one of the following:

§To map an account, select an account in the Major Accounts column, select one or more accounts in the Minor Accounts column, and click Add >.

Note:Select the asterisk (*) to map the entire major account.

To select multiple items at once, hold the Ctrl key and select the items. Alternatively, hold the Shift key to select a range of items.

§To unmap an account, select an account in the Mapped column and click < Remove.

 

6.Click Submit.

To modify the general ledger accounts mapped to cost object types for Cost Center Type:

Permission:Corporate Modify

Prerequisite:Cost object types and the chart of accounts must be configured.

Steps:

1.Select Corporate > COA Setup.

2.Click Coding Validation.

3.Click Cost Center Type under the Validation Type column.  

 

4.Click one of the cost center types under the Cost Center Type column.

 

5.Do one of the following:

§To map an account, select an account in the Major Accounts column, select one or more accounts in the Minor Accounts column, and click Add >.

Note:Select the asterisk (*) to map the entire major account.

To select multiple items at once, hold the Ctrl key and select the items. Alternatively, hold the Shift key to select a range of items.

§To unmap an account, select an account in the Mapped column and click < Remove.

 

6.Click Submit.

Configuring Smart Coding

Smart Coding automatically codes invoices based on either:

·the coding applied in previously saved approved or code-verified invoices, or

·smart coding rules that you import into OpenInvoice

There are two steps to configuring Smart Coding:

1.A user with the Corporate Admin permission configures the data elements that are available to Smart Code an invoice.

2.A user with the Corporate Partnership permission enables the Smart Coding function for suppliers.

How to Configure the Smart Coding Data Elements

Smart Coding uses the selected data elements (inputs) on an invoice line item and uses the selected data elements to decide the coding values to add (outputs) to the invoice line item.

Permission:Corporate Admin

Steps:

1.Select Corporate > Workflow Configuration.

2.Click Configure Smart Coding.

3.Select the data elements that are available for the automatic coding rules.

Note: Product/Service, Buyer Site, AFE Type, Cost Center Type, and OIR Activity Type (receipts only) are used to compare elements on previously approved invoices with the ready-to-be-coded document.

Supplier company is a system-required field and pre-selected for smart coding inputs.

Major, Minor and Subcode fields are system-required fields and pre-selected for smart coding outputs.

 

Note:The OIR Activity Type only applies to OI receipts using smart coding rules. The OI receipts functionality is obsolete and no longer supported. This option will be removed from the platform in a future release.

4.Click Submit.

How to Configure Smart Coding Static Rules

You can set static smart coding rules by importing Smart Coding rules into OpenInvoice. These rules will always override historical Smart Coding settings.

Prerequisite:Company Preferences > Populate Smart-Coding From Invoice must be set to Never 

Permission:Corporate Admin

Steps:

1.Click Export to download and save a copy of your existing Smart Coding rules in a file. The downloaded file will be in TSV format.

2.In the file, add new smart coding rules, modify or delete existing rules.

Note:Actions are Add, Modify, or Delete. For rules you do not want to change the Action column should remain Leave.

3.To upload a file to OpenInvoice, click Import, navigate to the file on your computer, and click Open.

Note:The uploaded file must be in TSV format.

How to Configure Smart Coding for Supplier Sites

You can modify the Smart Coding setting for supplier sites across multiple companies or within a single company.

To modify the Smart Coding setting for supplier sites across multiple companies:

Permission:Corporate Partnership

Prerequisite:The supplier site must be enabled for transactions.

Steps:

1.Select Corporate > Suppliers.

2.In the Suppliers section, click the Enabled counter.

Note:Apply a filter to narrow the list of supplier sites. Sort the list by clicking a column header.

3.Select the checkbox for each site.

Note:To select all sites, select the checkbox in the header.

4.Click the down arrow on the Select an Action button.

5.Select either Enable Smart Coding or Disable Smart Coding.

 

6.To modify this setting for supplier sites that are on another page in the list, navigate to the page and repeat steps 3 through 5.

To modify the Smart Coding setting for supplier sites within a single company:

Permission:Corporate Partnership

Prerequisite:The supplier site must be enabled for transactions.

Steps:

1.Select Corporate > Suppliers.

2.In the Suppliers section, click the Enabled counter.

Note:Apply a filter to narrow the list of supplier sites. Sort the list by clicking a column header.

3.Click the Supplier Site.

 

4.On the Supplier Configuration page, click Edit in the Early Pay Programs section.

5.In the Settings section select the Enable Smart Coding checkbox.

6.In the Supplier Sites section select the checkbox for each site to be modified.

Note:Select the checkbox in the list header to select all sites in the list.

All settings, including the Supplier Number, are applied to the selected sites.

7.Click Apply to Selected.

8.Click Done.

Supplier Coding

Supplier coding allows suppliers to enter general ledger coding on the tickets and invoices they submit. Suppliers only have the ability to enter the values that you provide to them. They do not have the ability to search through your chart of accounts.

How to Allow Supplier Coding for Sites in a Single Supplier Company

Permission:Corporate Partnership

Prerequisite:The supplier must be registered and enabled for transactions.

Steps:

1.Select Corporate > Suppliers.

2.In the Suppliers section, click the Enabled counter.

Note:Apply a filter to narrow the list of supplier sites. Sort the list by clicking a column header.

3.Click the Supplier Site name.

4.In the Early Pay Programs section, click Edit.

5.In the Settings section select the Allow Supplier Coding checkbox.

 

6.In the Supplier Sites section, select the checkbox for each site.

Note:To select all sites, select the checkbox in the header.

All settings, including the Supplier Number, will be applied to your selected sites.

7.Click Apply to Selected.

8.Click Done.
