Configuring Invoice Data Fields

Invoices have core fields that appear on every invoice. You can define the type of access that your users and your suppliers have to these core fields. Custom fields (called ADEs or Additional Data Elements) can be added to correspond to your company’s business practices. Users may populate your custom fields from a list of possible field responses that you define using the Corporate > Reference Data function in OpenInvoice. Custom fields and their possible responses will appear on your company’s OpenInvoice sites and the supplier OpenInvoice sites.

Note:OpenInvoice does not validate whether a field’s access setting is consistent with company preferences and other field configurations. To ensure invoices meet validation requirements to move through the workflow, consult your company’s current invoice content configuration and coding rules before configuring access.

Changes to the visibility of invoice data fields must be communicated to your suppliers to avoid non-compliance of any required fields. To send a message to one or more supplier sites, refer to Sending a Message to Suppliers. If an imported invoice, purchase order, or ticket specifies a value for an unmodifiable or hidden field, access configuration for that field will be ignored and the value will saved with the invoice and be included on the exports.

Invoice Element Types

·Restricted: fields displayed in the invoice header. Mandatory fields cannot be created. These fields can only be modified by a user with the Invoice Add Data permission.

·Header: fields displayed in the invoice header. Mandatory fields can be created.

·Line Item Detail: fields can only have one value per line item and cannot be the basis of a split allocation.

Note:The service date can be defined as a date range or a single date. If defined as a single date, OpenInvoice will assume that service was delivered beginning and ending on the same day.

·Line Item Coding: fields added to the coding records. General ledger codes can be mapped to some of these fields.

Note:Setting the AFE and Cost Center fields to None will also hide the property information on the invoice.

How to Add a Restricted Invoice Data Field

Permission:Corporate Admin

Prerequisite:If you are configuring a custom reference data type, the custom reference data types are entered by going to Corporate > Reference Data. See Managing Reference Data for more information.

Steps:

  1. Select Corporate > Workflow Configuration.

  2. Click Data Fields.

  3. Click the Restricted tab.

  4. Click Add Record to add a field.

    5.  

  5. Enter the field values.

  6. Click Submit.

Permission:Corporate Admin

Steps:

    4.  

  4. Click the Delete icon for the data field name you want to remove.

Note:Custom data fields that have been deleted will not be available on any new field tickets. Field Tickets created and submitted prior to the deletion of the custom field will still display the values and allows users to modify the field.

Data Field Details

Depending on the invoice element type, your field profile will include:

·Name: appears on the invoice and on invoice list pages

·Data Type: includes a selection of system responses as well as customized reference data. System choices include Date, Integer (whole numbers), Number (including decimals), and Text.

Note:The core Work Order field allows users to enter free text (Text) or to select from a drop-down of values (List) which are defined as customized reference data.

·Display Type: This field appears when you are using a custom data type. Choose between List or Searchable Text. Use the Searchable Text option for large sets of data where a pick list is not easy to use.  The Searchable Text field allows free text entry and also comes with a search function. The Searchable Text will be validated to ensure the entered text is valid and ‘Active’. Search results will only include valid and ‘Active’ records.

·Internal Code: code designated by your ERP.

·Sequence: order in which the custom fields appear on the invoice. The sequence numbers apply to all the invoice element types, even if some are in the header and some are on line items.

·Max Length: defined character limit for this data field. A max length is not required for Date (MM/DD/YYYY) or custom data types.

Note:If the core Work Order field’s data type has been changed from List to Text, the max length value will be considered invalid and cause errors.

·Required: 

§Please Select: makes the field optional.

§On Submit Only: makes the field mandatory for completion before a detail invoice can be submitted.

§On Code-Verify & On Approve: makes the field mandatory for completion before a detail invoice can be code verified.

§On Approve Only: makes the field mandatory for completion before a detail invoice can be approved.

§Always: makes the field mandatory at all times throughout the invoice’s workflow.

Note:If On Code-Verify & On Approve, On Approve, or Always is selected, all mandatory fields must contain values for the system to automatically code verify and/or automatically approve the invoice.

·Supplier Access: 

§None: cannot view or modify the data field.

§View: has read-only access to the data field.

§Modify: can enter data in the data field.

Note:Selecting None or View will prevent suppliers from being able to populate data in the field. For the AFE and Cost Center fields, the suppliers will not have access to the search controls.
Select Modify to allow suppliers to complete the required data field On Submit.

·Buyer Access:

§None: cannot view or modify the data field.

§View: has read-only access to the data field.

§Modify: can enter data in the data field.

·Unique Per Invoice: restricting to one or allowing more responses to populate the data field. This applies to Restricted and Header data types only. Selecting this checkbox will allow the user to only select one response. Deselecting this checkbox will allow the user to select more than one response.

·Editable: allows users with the Invoice Add Data permission to edit Restricted data types after they have been added. Deselecting this checkbox will only permit the original field creator or a user with the Invoice Admin permission to edit the field.

·Include on Notifications: specifies whether the field response should appear on email notifications.

·Exclude from Exports: specifies whether this field’s response should be included in the export file.

·Coding Validations: specifies whether the field can be mapped to the general ledger.

·Include in Smart Coding: specifies whether this field is included in the Smart Coding. This field only appears for custom reference data fields (also referred to as “flex fields”).

·Show Description in Allocation Summary: specifies whether the description for this custom reference data field appears in the Allocation Summary section of the Line Item on the Invoice.
