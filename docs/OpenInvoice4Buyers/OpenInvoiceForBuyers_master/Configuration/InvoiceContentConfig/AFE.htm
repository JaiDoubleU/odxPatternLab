AFE

AFEs and AFE types are added by going to Corporate > Reference Data. Buyer and supplier access to the AFE field is configured by going to Corporate > Workflow Configuration.

Note:When configuring invoice data fields, setting the AFE and Cost Center fields to None will also hide the property information on the invoice.

Relevant Company Preferences

·Require AFE or Cost Center

·PO Coding Override

How to Make the AFE a Required Field

These procedures require the Corporate Admin permissions.

Enable the Require AFE and Cost Center Fields Preference

Steps:

  1. Select Corporate > Company Preferences.

  2. Click Require AFE or Cost Center under the Invoice Content section.

  3. Select the Active checkbox.

    4. Note:This preference has no effect on B2B suppliers.

  4. Click Submit.

Configure GL Codes to Require an AFE

Use this procedure to configure certain general ledger account codes to require an AFE.

Steps:

  1. Select Corporate > COA Setup.

  2. Click Coding Validation.

  3. Click Account Type.

  4. Click AFE.

  5. Select the Major and Minor accounts you want to add.

    6.  

  6. Click Add To Required.

  7. Click Submit.

Configure GL Codes to Require Specific AFE Types

Steps:

  1. Select Corporate > COA Setup.

  2. Click Coding Validation.

  3. Click AFE Type.

  4. Click the AFE Type you want to configure.

    5.  

  5. Select the Major Accounts and Minor Accounts.

  6. Click Add >.

  7. Click Submit.

Configure Cost Centers to Require a PO

Steps:

  1. Select Corporate > Cost Centers.

  2. Search for a Cost Center Number.

    3. Note:To search for all Cost Center Numbers, type * in the Cost Center Number field and click Apply Filter.

     

  3. Click the Cost Center Number you want to configure.

  4. Select Required in the PO Required drop-down list.

  5. Click Submit.

Specify the Invoice Workflow Step That Requires an AFE

Steps:

  1. Select Corporate > Workflow Configuration.

  2. Click Invoice Data Fields.

  3. Click the Line Item Detail tab.

  4. Click AFE.

  5. Select the appropriate workflow.

    6.  

  6. Click Submit.

Reject Invoices Without an AFE

Steps:

  1. Select Corporate > Workflow Configuration.

  2. Click Configure Invoice Workflow Rules.

    3. Note:B2B invoices that are rejected by the invoice workflow rules will be automatically disputed on your behalf.

  3. Import the Invoice Workflow Rules file following the Invoice Workflow Rules Template.

How to Configure Supplier Access

This procedure configures whether a supplier can view or modify AFEs on an invoice.

Steps:

  1. Select Corporate > Workflow Configuration.

  2. Click Invoice Data Fields.

  3. Click the Line Item Coding tab.

  4. Click AFE.

  5. Select the appropriate Supplier Access option from the drop-down list.

  6. Click Submit.

How to Configure Buyer Access

This procedure configures whether users within your company are able to view or modify AFEs on an invoice.

  1. Select Corporate > Workflow Configuration.

  2. Click Invoice Data Fields.

  3. Click the Line Item Coding tab.

  4. Click AFE.

  5. Select the appropriate Buyer Access option from the drop-down list.

  6. Click Submit.

Validations Based on other Data Elements

·Restrict which AFEs can be specified on an invoice based on the selected buyer site by going to Corporate > AFEs.

·If an AFE is specified on a purchase order line and the PO Coding Override preference is turned off, that AFE must be on the invoice line referencing the purchase order line. If the PO Coding Override preference is enabled, the AFE defaulted from the purchase order can be modified.
