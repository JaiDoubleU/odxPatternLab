Invoice Workflow Rules

Invoice workflow rules determine how an invoice will be processed. Newly submitted and resubmitted invoices will be routed for processing in the OpenInvoice Platform, routed for processing in your back office, or automatically disputed (rejected). In addition, the rules determine whether invoices routed for processing in OpenInvoice will be automatically code verified and/or automatically approved.

The configured rules apply to detail and image invoices including invoices submitted via integration and the OpenInvoice Platform. If necessary, a user with the Invoice Admin permission can manually reroute an invoice being processed in the OpenInvoice Platform to your ERP; conversely, an invoice routed to your ERP can be unlocked so that it can be processed in the OpenInvoice Platform.

Components of an Invoice Workflow Rule

Workflow rules are configured using a TSV file. Each rule specified in the file consists of two components: the criteria considered to select the appropriate rule and the action to take.

Criteria to be considered may include the supplier type, cost objects, cost object validation, price book presence, price book alert, ticket presence, ticket alert, PO presence, PO alert, routing flag, and amount threshold. For a description of each workflow rule column, refer to Invoice Workflow Rules Template.

The action component of the rules allow you to specify how invoices will be processed and the error message that will appear if invoices are rejected. Action options include:

·Export (Client Workflow): the invoice is routed for processing in your company’s ERP and is included in the Submitted Invoice Export. The invoice is read-only in the OpenInvoice Platform. Messages can be added and suppliers can comment on the invoice, but an OpenInvoice action (Forward, Code, Approve, etc.) cannot be performed.

·Internal (OpenInvoice Workflow): the invoice is routed to the OpenInvoice platform for processing. The invoice is submitted to the Remit To site and Department as defined by the supplier and may be routed to a different recipient if invoice recipient routing rules are configured. On resubmit, the invoice is submitted to the owner who previously disputed the invoice.

·Reject (Auto-Dispute): direct entry, LIDO, Image OBO, and Image Scan invoices cannot be submitted if the platform rejects them based on the configured rules. B2B and Image Post invoices will be imported and submitted, but are then auto-disputed on your behalf. Your defined error message will be displayed in the application for direct entry, LIDO, Image OBO, or Image Scan invoices and is listed as a warning in the DOResponse document and dispute comment for B2B and Image Post invoices.

·Code Verify (Auto-Code Verification): the invoice will be automatically code verified if the invoice passes all coding business rules. If you are using the Code then Approve workflow the code verify rules are considered on invoice submission, the invoice is submitted to the Remit To site and Department as defined by the supplier and may be routed to a different recipient if invoice recipient routing rules are configured. If using the Approve then Code workflow the code verify rules are considered when the invoice receives final financial approval. The code verify action does not apply for the Approve Only workflow

·Approve (Auto-Approve): the invoice will be automatically financially approved if the invoice passes all validation business rules. If you are using the Code then Approve workflow the approve rules are considered when the invoice is code verified. If you are using the Approve Only or Approve then Code workflow processes the approve rules are considered on invoice submission, the invoice is submitted to the Remit To site and Department as defined by the supplier, and may be routed to a different recipient if invoice recipient routing rules are configured.

Things to Consider when Configuring Invoice Workflow Rules

·Invoices must match a configured rule exactly for the invoice to be processed by the rule’s action.

·Invoices match a rule with an Amount Threshold if the total invoice amount including taxes is less than or equal to the specified value. If the invoice is a credit, the absolute value of the invoice total must be less than or equal to the specified value.

·Rules entered as text in the TSV file are case-sensitive.

·The rules are evaluated sequentially from top to bottom. The first matching rule will determine the action and subsequent rules will not be evaluated. Therefore, the rules at the top of the file must be more specific than the rules at the bottom of the file.

·Each set of rules must include a default action (internal, export, or reject) to cover all conditions and ensure that all invoices are routed to the appropriate system for processing. This must be the last row in the file. For example, a final row with no conditions and Internal in the Action column will route all invoices not already routed to the OpenInvoice Platform using the preceding rules. The absolute minimum configuration is one row with an entry in the Action column.

·The invoice’s status and the sequence of processing of steps is determined by the workflow sequence being used.

How to Configure Invoice Workflow Rules

Permission:Corporate View to view rules

Corporate Admin to configure rules

Note:Existing rules will be replaced whenever a set of rules is imported. Any previous rule not included in a new import file will be deleted.

Steps:

  1. Select Corporate > Workflow Configuration.

  2. Click Configure Invoice Workflow Rules.

  3. Click Export to download and save the currently configured rules.

  4. In the file, enter the new workflow rules. Save the file in TSV format. For more information about the column headings, refer to Invoice Workflow Rules Template.

    5. Note:To disable the workflow rules, enter a single rule with Internal in the Action column and no other values.

  5. In OpenInvoice, click Import.

  6. Do one of the following:

    7. §To upload a file saved on your desktop, select File as the import method, click Choose File, navigate to the file on your computer, and click Open.

    §To copy the contents of a spreadsheet, select Copy/Paste as the import method, copy the information in your file, and paste it into the box.

     

  7. Click Upload.

How to Change the Workflow for Individual Invoices

Permission:Invoice Admin

Steps:

  1. Do one of the following:

    2. §Select Invoice > Dashboard and click a counter in the Invoices portlet.

    §Select Invoice > Dashboard and click the Invoice # in the My Ready to Approve portlet.

    §Perform a search by Invoice # using the Global Search field in the main navigation bar.

    §Perform a search with multiple criteria by going to Invoice > Invoice Search.

  2. Click the Invoice #.

  3. On the invoice toolbar, do one of the following:

§Click Unlock to allow users to process the invoice in the OpenInvoice Platform.

Note:If the invoice is pending export when it is unlocked, it will be removed from the Submitted Invoice Export job.

The invoice will remain in your ERP and must be manually deleted or otherwise resolved.

Changes made to the invoice in the ERP will not be reflected in the OpenInvoice version of the invoice.

§Click Export to make the invoice read-only in OpenInvoice and have the invoice sent to your ERP for processing.

Note:The invoice will be exported to your ERP the next time the Submitted Invoice Export scheduled job is run. The Export button is only available if the job has been scheduled.
