Automatic Code Verification

OpenInvoice can be configured to automatically mark invoices as code verified using the invoice workflow rules. Depending on your companyâ€™s workflow sequence, your code verification rules will be considered either upon invoice submission or the final financial approval.

Note:If you are using the Approve Only invoice workflow, Code Verify rules will be ignored.

OpenInvoice will code verify an invoice only if all associated business rules and coding validations are met. Invoices that are code verified by OpenInvoice are identified in the Invoice Journal by a Code Verified action completed by the System and a corresponding comment. You can report on invoices that were automatically code verified by searching for invoices code verified by the System.

Refer to Invoice Workflow Rules for more details about configuring automatic invoice code verification and system processing.

Invoice Auto-Code Verification and the Code then Approve Workflow

If you are using the Code then Approve invoice workflow, OpenInvoice will evaluate the Code Verify invoice workflow rules when the invoice is submitted. If the invoice qualifies for a Code Verify rule the invoice will be routed to the OpenInvoice Platform for processing and OpenInvoice will process any configured recipient routing rules. If all business rules and coding validations are met, the invoice will be code verified by the System and the invoice status will be Submitted. If the invoice is not code verified by OpenInvoice, a user must code verify the invoice.

Invoice Auto-Code Verification and the Approve then Code Workflow

If you are using the Approve then Code invoice workflow, OpenInvoice will evaluate the Code Verify invoice workflow rules when the invoice receives final financial approval (either by a user clicking the Approve button on an invoice or automatically based on the invoice workflow rules). If the invoice qualifies for a Code Verify rule and all business rules and coding validations are met, the invoice will be code verified by the System and the invoice status will be changed to Approved. If the invoice is not code verified by OpenInvoice, a user must code verify the invoice.
