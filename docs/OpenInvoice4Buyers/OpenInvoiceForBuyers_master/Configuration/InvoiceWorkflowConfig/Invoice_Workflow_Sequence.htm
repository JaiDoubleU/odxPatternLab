Invoice Workflow Sequence

OpenInvoice supports three invoice workflow processes, each with a predefined sequence of events:

·Code then Approve: requires code verification before financial approval.

·Approve Only: allows financial approval without a separate code verification step.

·Approve then Code: allows financial approval before coding is verified.

Your chosen invoice workflow is configured by OpenInvoice Support and applies to all invoices processed in the OpenInvoice Platform. The default workflow is Code then Approve.

In the Code then Approve and Approve Only workflow processes, the invoice status changes to Approved when the invoice receives final financial approval. In the Approve then Code workflow, the invoice status changes to Approved when the invoice is code verified.
