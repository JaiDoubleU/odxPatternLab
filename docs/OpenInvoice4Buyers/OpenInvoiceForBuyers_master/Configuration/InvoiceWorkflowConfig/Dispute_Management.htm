Dispute Management

Invoice dispute routing rules determine whether a disputed invoice will be routed to a department within your company or returned to the supplier or indexer for resolution. The configured rules apply to all disputed invoices regardless of how the dispute was initiated (auto-disputed, manually disputed, or disputed by the integrated invoice response). If an invoice does not match any of the configured rules, or there are no rules configured, then the invoice will be disputed to the supplier or indexer for resolution.

Internal Dispute Considerations

When invoices are routed to an internal department for dispute resolution:

·The Invoices portlet will include a Disputes to Resolve counter, which is only visible to users who have the Invoice Modify permission in the internal dispute department.

·The invoice’s Invoice Journal will display an action of Disputed Internally with the comment provided by the user who disputed the invoice. The action and comment are not visible to the supplier.

·The invoice status does not change while the invoice is reviewed by the internal dispute department. The status will only change to Disputed if a member of the internal dispute department with the Invoice Modify permission disputes the invoice to the supplier or indexer for resolution. The time the internal dispute department spent reviewing the invoice does not count towards the amount of time it takes the supplier or indexer to resubmit the invoice.

·Users with the Invoice Modify (IM) permission can cancel an Image OBO invoice.

·Users with the Invoice Delete (IM) permission can delete an Image Post invoice.

Note:Image Post invoices can only be disputed if there is a configured rule applying to the invoice.

Dispute Routing Rules

Each rule consists of two components: the criteria considered to select the appropriate rule and the action to take. When creating the criteria for rule selection, consider the following:

·Criteria on disputed invoices must match a configured rule exactly for routing to occur.

·Rules entered as text in the TSV file are not case-sensitive.

·The rules are evaluated sequentially from top to bottom. The first matching rule will determine the action and subsequent rules will not be evaluated. Therefore, the rules at the top of the file must be more specific than the rules at the bottom of the file.

Routing Rule Options

Disputed invoices are internally routed based on:

·Invoice Site: the site that owned the invoice when it was disputed.

·Invoice Origin: enter Scan and Index for Image OBO and Image Post invoices, Integration Supplier for B2B invoices, or Online Supplier for Direct Entry, Image Scan, and LIDO invoices.

·Dispute Site: the site that the invoice will be routed to for dispute resolution.

·Dispute Department: the department within the Dispute Site that the invoice will be routed to for dispute resolution.

How to Configure Dispute Routing Rules

Permission:Corporate View to view rules

Corporate Admin to configure rules

Note:Existing rules will be replaced whenever a set of rules is imported. Any previous rule not included in the new import file will be deleted.

Steps:

  1. Select Corporate > Workflow Configuration.

  2. Click Configure Dispute Routing Rules.

  3. Click Export to download and save the currently configured rules.

  4. In the file, enter the new routing rules. Save the file in TSV format. For more information about the column headings, refer to Invoice Dispute Routing Rules Template.

    5. Note:To disable dispute routing, create a file with no rules.

  5. In OpenInvoice, click Import.

  6. Do one of the following:

    7. §To upload a file saved on your desktop, select File as the import method, click Choose File, navigate to the file on your computer, and click Open.

    §To copy the contents of a spreadsheet, select Copy/Paste as the import method, copy the information in your file, and paste it into the box.

     

  7. Click Upload.
