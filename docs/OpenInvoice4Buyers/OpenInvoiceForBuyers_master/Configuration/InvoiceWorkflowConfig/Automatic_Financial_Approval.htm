Automatic Financial Approval

OpenInvoice can be configured to automatically mark invoices as financially approved using the invoice workflow rules. Depending on your companyâ€™s workflow sequence, your financial approval rules will be considered either upon invoice submission or code verification.

OpenInvoice will financially approve an invoice only if all associated business rules and coding validations have been met. Invoices that are financially approved by OpenInvoice are identified in the Invoice Journal by an Approve action completed by the System and a corresponding comment. You can report on invoices that were automatically approved by searching for invoices approved by the System.

Refer to Invoice Workflow Rules for more details about configuring automatic invoice approval and system processing.

Invoice Auto-Approval and the Code then Approve Workflow

If you are using the Code then Approve invoice workflow, OpenInvoice will evaluate the Approve invoice workflow rules when the invoice is code verified (either by a user clicking the Verify Coding or the Verify and Approve buttons on an invoice or automatically based on the invoice workflow rules). If the invoice qualifies for an Approve rule and all business rules and coding validations are met, the invoice will be financially approved by the System and the invoice status will be changed to Approved. If the invoice is not financially approved by OpenInvoice, the user must forward the invoice for approval.

Invoice Auto-Approval and the Approve Only Workflow

If you are using the Approve Only invoice workflow, OpenInvoice will evaluate the Approve invoice workflow rules when the invoice is submitted. If the invoice qualifies for an Approve rule the invoice will be routed to the OpenInvoice Platform for processing and OpenInvoice will process any configured recipient routing rules. If all business rules and coding validations are met, the invoice will be financially approved by the System and the invoice status will be changed to Approved. If the invoice is not financially approved by OpenInvoice, a user must approve the invoice.

Invoice Auto-Approval and the Approve then Code Workflow

If you are using the Approve then Code invoice workflow, OpenInvoice will evaluate the Approve invoice workflow rules when the invoice is submitted. If the invoice qualifies for an Approve rule the invoice will be routed to the OpenInvoice Platform for processing and OpenInvoice will process any configured recipient routing rules. If all business rules and coding validations are met, the invoice will be financially approved by the System, the invoice status will be Submitted, and the invoice will be ready for coding. If the invoice is not financially approved by OpenInvoice, a user must approve the invoice.
