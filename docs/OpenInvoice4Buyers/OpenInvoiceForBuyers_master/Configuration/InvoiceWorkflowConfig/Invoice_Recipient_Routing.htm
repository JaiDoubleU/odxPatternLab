Invoice Recipient Routing

Invoice recipient routing rules allow invoices processed in the OpenInvoice Platform to be routed to:

·A specific site and department.

·A specific site, department, and user.

·A requisitioner.

Invoices are submitted to the Remit To site and department as specified on the invoice by the supplier. Invoice recipient routing, performed only on initial invoice submission, forwards the invoice to a recipient who then becomes the invoice owner.

Note:If the invoice is disputed and then resubmitted, it will be resubmitted to the user who disputed it.

Recipient Routing Rules

Each rule consists of two components: the criteria used to select the appropriate rule and the routing destination. When creating the criteria for rule selection, consider the following:

·Criteria on submitted invoices must match a configured rule exactly for routing to occur.

·The rules are evaluated sequentially from top to bottom. The first matching rule will determine the recipient and subsequent rules will not be evaluated. Therefore, the rules at the top of file must be more specific than the rules at the bottom of the file.

·If the first matched routing specifies a destination that is no longer valid, the invoice is not routed. Subsequent rules are not evaluated.

·A default rule is not required. If no rules match or if the Remit To site and Department are the same as the routing destination determined by the rules, the invoice is not routed. The invoice remains in the Remit To site and Department as selected by the supplier.

·A Department is a valid recipient in the Code then Approve workflow if there is at least one user in the department with the Invoice Modify permission. A Department is a valid recipient in the Approve Only or Approve then Code workflow processes if there is at least one user in the department with either the Invoice Modify permission or Invoice Approve permission.

·You must provide suppliers with this routing code. If the supplier enters a routing code that is not configured and there are no other criteria in the rule, the invoice is not routed.

·When defining the routing code, email addresses may be used. Spaces are permitted but will be ignored (the routing code for Send this to Jesse is sendthistojesse). If the routing code includes a comma, the text in the TSV file must be copied and pasted into the Paste field. If uploaded in a file, quotation marks will appear around the routing code because it contains a comma. The supplier must then input the routing code in quotation marks for the OpenInvoice platform to recognize it.

·A user is a valid recipient if they have the Invoice View permission in the destination site and department.

·When specifying a user as the valid recipient, the text in the TSV file must be copied and pasted into the Paste field. The rules cannot be uploaded in an Excel TSV file. Excel will put quotation marks around the user’s name because the format contains a comma. The entry will then be rejected for not being formatted correctly.

·If routing to a requisitioner, all other values on the rule must be blank.

·If multiple requisitioners are listed on an invoice, the invoice is routed to the first requisitioner listed. If the invoice cannot be routed to the first requisitioner, the invoice is not routed. Any other requisitioners on the invoice are not considered.

§If the first requisitioner on the invoice belongs to one site and multiple departments:

-The invoice is routed to the department where the requisitioner has the highest permission (an invoice would be routed to the department where the user has the Invoice Code permission over the department where the user has the Invoice Modify permission).

-If the user has the highest permission in multiple departments, including the Remit To department specified by the supplier, the invoice is routed to the requisitioner in the Remit To department specified by the supplier.

-If the user has the highest permission in multiple departments but is not a member of the Remit To department specified by the supplier, the invoice is routed to the requisitioner in the first department in alphabetical order.

§If the requisitioner belongs to multiple sites:

-If the user is a member of the Remit To site and Department, the invoice is routed to the requisitioner in the Remit To site and Department.

-Otherwise, the invoice is not routed.

How to Configure Recipient Routing Rules

Permission:Corporate View to view rules

Corporate Admin to configure rules

Note:Existing rules will be replaced whenever a set of rules is imported. Any previous rule not included in a new import file will be deleted.

Steps:

  1. Select Corporate > Workflow Configuration.

  2. Click Configure Invoice Recipient Rules.

  3. Click Export to download and save the currently configured rules.

  4. In the file, enter the new routing rules. Save the file in TSV format. For more information about the column headings, refer to Invoice Recipient Routing Rules Template.

    5. Note:To disable workflow routing, create a file with no rules.

  5. In OpenInvoice, click Import.

  6. Do one of the following:

    7. §To upload a file saved on your desktop, select File as the import method, click Choose File, navigate to the file on your computer, and click Open.

    §To copy the contents of a spreadsheet, select Copy/Paste as the import method, copy the information in your file, and paste it into the box.

     

  7. Click Upload.
