Configure Field Ticket Data Fields

Custom data fields can be created for field tickets. When configuring a new data field you can specify its label and data type. The data type includes text, number, date, or values from a validated list of options. The validated list can either be in the form of a drop-down list or searchable text. Validated lists can be manually administered within OpenInvoice or electronically using an API. Each data field can be set to be unique to a field ticket and even a field ticket type. Custom data fields can be used in the ticket search or when configuring your ticket stacks. They are available on the summary and detailed reports.

How to Add a Field Ticket Data Field

Permission:Corporate Admin

Prerequisite:If you are configuring a custom reference data type, the custom reference data types are entered by going to Corporate > Reference Data.

Steps:

  1. In OpenInvoice, select Corporate > Workflow Configuration.

  2. Click Data Fields.

    3.  

  3. Click the Header tab.

  4. Click Add Record to add a field.

  5. In the Available On drop-down list, select Ticket.

    6. Note:The Data Field page refreshes with the fields applicable to field tickets. If you select Invoice and Ticket, all the fields display.

     

  6. Select the Ticket Type.

  7. Enter the required field values (highlighted in yellow).

  8. Select the appropriate Supplier Access from the drop-down list.

  9. Click Submit.

Data Field Details

Depending on the data element type, your field profile will include:

·Name: appears on the field ticket and on field ticket list pages

·Data Type: includes a selection of system responses as well as customized reference data. System choices include Date, Integer (whole numbers), Number (including decimals), and Text.

Note:The core Work Order field allows users to enter free text (Text) or to select from a drop-down of values (List) which are defined as customized reference data.

·Display Type: This field appears when you are using a custom data type. Choose between List or Searchable Text. Use the Searchable Text option for large sets of data where a pick list is not easy to use. The Searchable Text field allows free text entry and also comes with a search function. The Searchable Text will be validated to ensure the entered text is valid and ‘Active’. Search results will only include valid and ‘Active’ records.

·Internal Code: code designated by your ERP.

·Sequence: order in which the custom fields appear on the invoice. The sequence numbers apply to all the invoice element types, even if some are in the header and some are on line items.

·Max Length: defined character limit for this data field. A max length is not required for Date (MM/DD/YYYY) or custom data types.

Note:If the core Work Order field’s data type has been changed from List to Text, the max length value will be considered invalid and cause errors.

·Available On: Select Invoice, Ticket, or Invoice and Ticket.

·Ticket Type: Select the appropriate ticket type: General, Water Hauling, or Labor, Equipment, Material.

·Required For Ticket: 

§Please Select: makes the field optional.

§On Submit: makes the field mandatory for completion before a detail field ticket can be submitted.

§On Approve: makes the field mandatory for completion before a detail field ticket can be approved.

Note:If On Approve is selected, all mandatory fields must contain values for the system to automatically code verify and/or automatically approve the field ticket.

·Supplier Access: 

§None: cannot view or modify the data field.

§View: has read-only access to the data field.

§Modify: can enter data in the data field.

Note:Selecting None or View will prevent suppliers from being able to populate data in the field. For the AFE and Cost Center fields, the suppliers will not have access to the search controls.
Select Modify to allow suppliers to complete the required data field On Submit.
