Content Model Configuration

The content model determines which fields are read-only, optional entry, or required entry when submitting and approving tickets. Your company will need to consult with OpenInvoice Support to create your content model for use in the OpenInvoice Platform.

Note:There are core application fields that are not configurable.

Content models define which fields are present:

·when a supplier submits a field ticket

·when the buyer approves a pending field ticket

Configuration Process

Your company will submit to OpenInvoice Support a document indicating the desired content model configuration. OpenInvoice Support configures the fields in the platform and sets up the validation rules. The available options for each configurable field are:

·Required: 

§Not Required: makes the field optional.

§On Submit Only: makes the field mandatory for completion before a ticket can be submitted.

§On Approve Only: makes the field mandatory for completion before a detail invoice can be approved.

·Supplier Access: 

§None: cannot view or modify the data field.

§View: has read-only access to the data field.

§Modify: can enter data in the data field.

Note:Selecting None or View will prevent suppliers from being able to populate data in the field. For the AFE and Cost Center fields, the suppliers will not have access to the search controls.
Select Modify to allow suppliers to complete the required data field On Submit.

·Buyer Access:

§None: cannot view or modify the data field.

§View: has read-only access to the data field.

§Modify: can enter data in the data field.

Field Ticket Data

Buyers can configure the fields available for suppliers to populate on a ticket. Suppliers will see ticket fields for each buyer based on the buyer’s configuration settings. Suppliers submitting through an API will be provided with each buyer’s field configuration when being on-boarded for the specific buyer. To learn more about configuring fields to hide/display on tickets go to Configuration Process.

The following fields are available for content model configuration on tickets:

Header Data

·AFE: provided by the requisitioner. The AFE can be entered at the header level for common coding (when the coding is the same for the entire ticket).

·Cost Center: provided by the requisitioner. The Cost Center can be entered at the header level for common coding (when the coding is the same for the entire ticket).

·Requisitioner: buyer contact who ordered the product or service. The requisitioner must be a valid user within OpenInvoice who can approve a field ticket (Field Ticket Modify).

·Currency: code for the currency (required).

·Driver: name of the driver for water hauling tickets.

·Location: the location of the service.

·Office Approver: the name of the person responsible for approving the invoice associated with the field ticket. If the field ticket approver includes the Office Approver on the field ticket it may speed up the process of routing the invoice to the correct approver.

·Superintendent: the name of the superintendent, who is an employee of the oil company.

·Truck Number: the number of the truck for water hauling and labor, equipment, material tickets.

·Truck Capacity: the capacity of the truck for water hauling tickets.

Line Data

·Location: location where the product was delivered or the service was performed.

·AFE: provided by the requisitioner. The AFE can be entered at the header level for common coding (when the coding is the same for the entire ticket). The AFE must be valid for the buyer.

·Cost Center: provided by the requisitioner. The Cost Center can be entered at the header level for common coding (when the coding is the same for the entire ticket). The Cost Center must be valid for the buyer.

·Disposal Location: the disposal location for water hauling tickets.

·Disposal Ticket: the ticket number associated with the disposal.

·Gauge Start Unit: the initial reading of the gauge in feet for a water hauling ticket

·Gauge Start Subunit: the initial reading of the gauge in inches for a water hauling ticket

·Gauge Finish Unit: the final reading of the gauge in feet for a water hauling ticket

·Gauge Finish Subunit: the final reading of the gauge in inches for a water hauling ticket

·Hours Worked Unit: the number of hours worked for a water hauling ticket

·Hours Worked Subunit: the number of minutes worked for a water hauling ticket

·Payload: amount of payload for water hauling tickets.

·PO: purchase order number.

·PO Line: the appropriate line number on the purchase order.

·Total: the total for the line item.

·GL Coding: provided by the requisitioner. The GL Coding must be valid for the buyer.

·Quantity: number of units bought.

·Unit of Measure: unit of measure for the product or service provided.

·Rate: rate charged for each unit of the product or service.

·Volume: the volume for water hauling tickets. The unit of measure for the volume must be selected from the drop-down list.

Field Ticket Journal

·Submitted: ticket has been submitted by the supplier to the buyer.

·Approved: ticket has been approved by the buyer.

·Unapproved: ticket has been unapproved by the buyer.

·Disputed: if a ticket has a red compliance alert it can be automatically disputed. Tickets that are manually disputed require a comment to be added to explain the reason for the dispute.

·Cancelled: ticket has been cancelled by the supplier.

Configurable Details

OpenInvoice allows buyers to configure the required data elements for tickets received from suppliers. Configurable fields on tickets include:

·AFE

·Cost Center

·Major / Minor / Subcode

·Requisitioner

·Location
