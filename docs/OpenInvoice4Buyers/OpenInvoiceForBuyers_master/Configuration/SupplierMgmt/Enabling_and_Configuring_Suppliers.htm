Enabling and Configuring Suppliers

Note:Supplier sites can be auto enabled for transactions. To use this feature, a supplier number must be entered at the time of submitting an on-ramping request.

How to Enable a Registered Supplier for the First Time

Permission:Corporate Partnership

Note:If enabling a registered supplier site with a supplier number combination that already exists for an unregistered supplier site, you will be prompted to merge the supplier profiles.

If viewing a supplier site that is eligible to be merged based on a matching supplier number combination, a message will direct you to the Merge Supplier Wizard to merge the supplier profiles.

For more information about merging supplier profiles, refer to Merge Registered and Unregistered Supplier Records.

Steps:

  1. Select Corporate > Suppliers.

  2. In the Tasks section, click Enable Supplier.

  3. Perform a search for the supplier.

    4. Note:If criteria is unknown, type an asterisk (*) into one of the fields to return a complete list.

  4. Select the radio button next to the supplier you want to enable for transactions.

  5. Scroll down to the bottom of the page and click Submit.

  6. Select the invoice submission method(s).

    7. Note:To select all invoice methods, select the Transactions Enabled checkbox.

    You can also configure and apply other supplier settings at this time.

    If more than one site exists for the company, check the other supplier site(s) you wish to enable for transactions.

     

  7. Click Submit or Apply to Selected.

Note:Registered suppliers will receive an email informing them that they have been enabled and can begin transacting with your company. Enabling a supplier for invoice automatically enables the supplier for field ticket as well.

How to Modify Supplier Payment Terms

Permission:Corporate Partnership

Steps:

  1. Select Corporate > Suppliers.

  2. In the Tasks section, click Search Suppliers.

  3. Enter search criteria.

  4. Click Apply Filter.

  5. Select a Supplier Site from the search results.

  6. Click the Edit button under the Early Pay Programs section.

  7. Select the Custom option for Invoice Due Date and enter the number of days after the Invoice Submission Date.

    8.  

  8. Select the checkbox next to the site(s) for which this change will apply.

  9. Click Done.

How to Copy Configuration Settings From an Existing Supplier Company

Permission:Corporate Partnership

Note:Settings for Priority Pay, Allow Future Billing, Allow Supplier Coding, Allow External Field Ticket Approval, Enable Smart Coding, Allow External PO, Selected Early Pay Programs, and Invoices Require Early Pay Program will be copied. Settings can be modified after copying.

Steps:

  1. Select Corporate > Suppliers.

  2. Do one of the following:

    3. ยงIf the supplier is enabled, click the Enabled counter in the Suppliers section.

    ยงIn the Tasks section, click Search Suppliers.

  3. If necessary, apply a filter to narrow the list.

  4. Click the supplier site.

  5. On the Supplier Configuration page, click Edit below the Settings and Early Pay Programs sections.

  6. Enter the supplier number and secondary supplier number of the company you wish to copy configuration settings from.

    7.  

  7. Click outside the Supplier Number (or Secondary Supplier Number) field and the Load Configuration button will be enabled.

    8. Note:In order for the Load Configuration button to be enabled, all suppliers with the entered supplier number and secondary supplier number must have the same configuration. If the supplier number and secondary supplier number refer to two (or more) sites that have different configurations, the Load Configuration button will remain grayed out.

  8. Click Load Configuration to copy settings.

  9. If necessary, modify the copied settings.

  10. Click Submit.

How to Delete Unregistered Supplier Sites

Permission:Corporate Partnership

Prerequisite:A user with the Invoice Modify (IM) permission must modify the supplier on all invoices associated with the site you want to delete.

Note:To delete unregistered supplier sites via file upload, refer to Manage Unregistered Supplier Sites via File Upload.

If you are deleting a supplier site that you have created in error, all invoices associated with the site must be corrected and moved to another supplier site.

Steps:

  1. Select Corporate > Suppliers.

  2. In the Tasks section, click Search Suppliers.

  3. In the Company Type drop-down, select Unregistered.

  4. Click Apply Filter.

  5. Click the Supplier Site name.

  6. Under the Supplier Site Details section, click Delete.

    7.  

  7. Click Yes to confirm.

How to Assign Early Pay Programs to a Single Supplier

Permission:Corporate Partnership

Prerequisite:The supplier is registered and enabled.

There is at least one active Early Pay program.

Note:You can assign Early Pay programs to a supplier during the enablement process.

Steps:

  1. Do one of the following:

    2. ยงIn the Tasks section, click Search Suppliers and perform a search for enabled suppliers.

    ยงIn the Suppliers section, click the Enabled counter.

  2. If necessary, apply a filter to narrow the list.

  3. Click the supplier site.

  4. On the Supplier Configuration page, click Edit in the Early Pay Programs section.

  5. Select one or more Early Pay programs from the Available Programs box.

    6. Note:To select multiple items at once, hold the Ctrl key and select the programs. Alternatively, hold the Shift key to select a range of programs.

    Each program assigned to must have a different discount percentage.

     

  6. Click Add to move the program(s) into the Selected Programs box.

  7. Select the Invoices Require Early Pay Program checkbox if you require the supplier to specify an Early Pay program on all of the siteโs invoices.

  8. Do one of the following:

ยงIf the supplier has just one site, click Submit.

ยงIf the supplier has multiple sites, select the checkbox next to each site you want to assign the programs and click Apply to Selected and then click Done.

How to Assign Early Pay Programs to Multiple Suppliers

Permission:Corporate Partnership

Prerequisite:The supplier is registered and enabled.

There is at least one active Early Pay program.

Supplier Configuration Details

Users with the Corporate Partnership permission can configure supplier site details, Remit To information, and general settings on-behalf of your unregistered suppliers. Users with both the Corporate Partnership and Payment Admin permission can configure payment information on behalf of your unregistered suppliers.

ยทSupplier Site Details: company name and site name can be modified for an unregistered supplier. The site can be deleted if there are no invoices associated with the site.

ยทFirst and Last Submit Date: view the first and last submit date for direct entry invoices and field tickets.

ยทBanking Details: account number, type, and payment method can be entered and/or modified for an unregistered supplier

ยทRemit To Address and Remit To Contact: 

ยงValidate for Payment: 

-For registered suppliers, this field displays Yes if the companyโs banking information has been entered, is validated, or is in the process of being validated. If No is displayed, the company has not entered their banking information, has not asked for the information to be validated, or the validation failed.

-For unregistered suppliers, this field is a checkbox that you will select when the supplier is ready to be validated for payment. The Pay-To Name and Remit To Address fields must be completed and the supplier must be enabled for transactions.

ยทPayment Terms: specific payment schedule for this supplier

ยงCompany Default: payments to the supplier will be scheduled using the settings specified in Payment > Payment Terms.

ยงNever: payments to the supplier will not be scheduled in OpenInvoice.

ยงCustom: payments to the supplier will be scheduled the specified number of days after either the invoice submission date or the invoice date

ยทSettings:

ยงSupplier Number: required if the Transactions Enabled checkbox is selected.

ยงSecondary Supplier Number: optional number that may group a set of sites

Note:The combination of supplier number and secondary supplier number must be unique to the supplier company. More than one site within the supplier company can share the same combination.

The supplier number and secondary supplier number combination cannot be modified if the site is part of a new or in-progress supplier merge request.

ยงSupplier Type: only appears if the company preference Enable Supplier Categorization is enabled.

ยงTransactions Enabled: selected once you are ready to start transacting with the supplier.

-Enable Details and B2B Invoices: allows a registered supplier to create and submit detailed invoices using direct entry, LIDO, or integration (B2B).

-Enable Buyer Scanned Invoices: allows users from your company to create and submit invoices on behalf of the supplier (Image OBO).

-Send Invoice Processing Summary: automatically sends the supplier an email notification every time a user from your company has created and submitted an invoice on their behalf. The email includes a summary of the invoice as well as an invitation to register with OpenInvoice.

-Enable Supplier Scanned Invoices: allows a registered supplier to create and submit invoices from a scanned image that is uploaded into OpenInvoice (Image Scan).

ยงPriority Pay: also known as Quick Pay, marks supplier invoices with a QP alert and prioritizes these invoice to the top of an invoice list page. Priority Pay agreements are made outside of the OpenInvoice Platform and, as such, remit amounts calculated in OpenInvoice are not adjusted.

Note:Priority Pay is different from Early Pay price books which are documented in OpenInvoice. Early Pay discounts are calculated by OpenInvoice and the number of days until the discount period expires is displayed in the interface.

ยงAllow Future Billing: By default, the invoice line Service Date (or Service Date From if using a range) value must be on or before the invoice date. If this is selected, the invoice line Service Date (or Service Date From if using a range) value can be later than the invoice date.

ยงAllow Supplier Coding: applies to registered suppliers only. If selected, the supplier will have three additional fields when entering invoices in the platform where the supplier can enter your companyโs major, minor, and subcodes.

Note:You must provide the supplier with your general ledger codes.

If an integrated supplier imports an invoice with coding and the Allow Supplier Coding setting is not enabled, the coding will be removed before the invoice is submitted.

ยงAllow External Field Ticket Approval: allows a registered supplier, using PIDX field ticket creation, to have their OIR charges automatically approved when an approver name, approval date, and other required fields are submitted with OIR charges. This setting is obsolete as of August 2016.

Note:The company preference 2 Step Approval Workflow must be enabled.

ยงEnable Smart Coding: automatically applies coding based on past approved or code verified invoices based on your company preference on how you populate smart coding on an invoice. Smart Coding can be used if the coding is not being applied by a supplier or from another document.

ยงAllow External PO: By default, if valid purchase orders exist in OpenInvoice for the supplier, the referenced purchase order must be one of the purchase orders in platform. If this is selected, the supplier is able to enter any value in the purchase order field.

ยทEarly Pay Programs

Note:Early Pay programs can be assigned to multiple supplier sites from the Supplier Search list page. For instructions on how to create Early Pay programs, refer to Configuring Early Pay Programs.

ยงAvailable Programs: Early Pay programs that are active but not assigned to the supplier site.

ยงSelected Programs: Early Pay programs assigned to the supplier site.

ยงInvoices Require Early Pay Program: select if you want to require the supplier to apply an Early Pay program on every invoice submitted from this site.
