Invoice Workflow Rules Template

The rules from the imported file will replace all Invoice Workflow rules in the system. Ensure that the import file contains all of the rules that have been defined, in the order that they must be evaluated. The Configure Invoice Workflow Rules template can be accessed from the Corporate > Workflow Configuration page.

Each row of the spreadsheet represents one rule, consisting of two components: the criteria considered to select the appropriate rule and the action to take. An invoice must match a rule exactly for the invoice to be processed by the rule’s action.

Note:All values entered in the fields below are not case-sensitive (acceptable values include: export, Export, and EXPORT).      

      

Column

Required Field

Description

Supplier Type

 

Enter one of the supplier type values defined in Reference Data for the rule to consider invoices when the supplier type matches the invoice supplier site’s supplier type.

Leave the cell blank if the supplier type is irrelevant for the rule.

Supplier Number (primary)

 

Enter a supplier number for the rule to apply to any supplier site assigned that value.

Enter “No Value” to associate the rule with supplier sites that are not assigned a supplier number.

Leave the cell blank if supplier number is irrelevant for the rule.

Secondary Supplier Number

 

Enter a secondary supplier number for the rule to apply to any supplier site assigned that value.

Enter “No Value” to associate the rule with supplier sites that are not assigned a secondary supplier number.

Leave the cell blank if secondary supplier number is irrelevant for the rule.

AFE

 

Enter Yes for the rule to consider invoices that have a valid AFE reference on all lines.

Enter No for the rule to consider invoices with at least one line that does not have a valid AFE reference.

Leave the cell blank if AFEs are irrelevant for the rule.

CC

 

Enter Yes for the rule to consider invoices that have a valid cost center reference on all lines.

Enter No for the rule to consider invoices with at least one line that does not have a valid cost center reference.

Leave the cell blank if cost centers are irrelevant for the rule.

CC is Valid for AFE

 

Enter Yes for the rule to consider invoices if the specified cost center is valid for the specified AFE.

Enter No for the rule to consider invoices with at least one line where the specified cost center is not valid for the specified AFE.

Leave the cell blank if validity is irrelevant for the rule.

Price Book Presence

 

Enter Yes for the rule to consider invoices that have price book references.

Enter No for the rule to consider invoices that do not have price book references.

Leave the cell blank if the presence of a price book is irrelevant for the rule.

Price Book Alert

 

Enter Red, Amber, Green, or Blue for the rule to consider invoices based on a specific alert severity.

Leave the cell blank if the alert is irrelevant for the rule.

Ticket

 

Enter Yes for the rule to consider invoices that have ticket references.

Enter No for the rule to consider invoices that do not have ticket references.

Leave the cell blank if the presence of tickets is irrelevant for the rule.

Ticket Approved

 

Enter Yes for the rule to consider invoices that have approved tickets.

Enter No for the rule to consider invoices that do not have approved tickets.

Leave the cell blank if the presence of approved tickes is irrelevant for the rule.

Ticket Alert

 

Enter Green, Amber, or Red for the rule to consider invoices based on a specific alert severity.

Leave the cell blank if the alert is irrelevant for the rule.

PO Presence

 

Enter Yes for the rule to consider invoices with purchase order references. The referenced purchase order does not need to be in the OpenInvoice Platform (you may reference a purchase order in your back office system).

Enter No for the rule to consider invoices without purchase order references.

Leave the cell blank if the presence of purchase order purchase orders is irrelevant for the rule.

PO Alert

 

Enter Green, Amber, or Red for the rule to consider invoices based on a specific alert severity.

Leave the cell blank if the alert is irrelevant for the rule.

Routing Flag

 

Enter OpenInvoice Processing or Client Processing for the rule to consider invoices based on the flag in the purchase order. The corresponding value is specified on the purchase order at the time the purchase order is submitted to the OpenInvoice Platform.

Leave the cell blank if the flag is irrelevant for the rule.

Amount Threshold

 

Enter the maximum total invoice amount (includes taxes) that will qualify for the rule.

Credit invoices qualify for the rule if the absolute value of the total invoice amount is less than or equal to the amount threshold.

Leave the cell blank if the total invoice amount is irrelevant for the rule.

Action

X

Enter Internal, Export, Reject, Code Verify, or Approve as the action to be performed by the rule.

Error Message

 

Enter the message to be displayed for each rule that has Reject as the action.

Messages containing a comma will be displayed in quotation marks if uploading an Excel TSV file. Quotation marks will not appear if the values are copied and pasted into the Paste field.

Routing Code

 

Routing Code can be assigned with the Internal, Code Verify, or Approve rule actions. The Approve rule only uses the routing code if the workflow preference is “Approve then Code.”

The Routing Code will only be assigned if the invoice is submitted for the first time.

      

Invoice Recipient Routing Rules Template

The rules from the imported file will replace all Invoice Recipient rules in the system. Ensure that the import file contains all of the rules that have been defined, in the order that they must be evaluated. The Configure Invoice Recipient Rules template can be accessed from the Corporate > Workflow Configuration page.

No fields are required (a blank template can be imported); however:

·if ROUTE_REQUISITIONER is TRUE, all other fields must be blank.

·if an attribute on the invoice (SUPPLIER_NAME, SUPPLIER_SITE_NAME, REMIT_SITE_NAME, REMIT_DEPT_NAME, or ROUTING_CODE) is specified, you must specify the destination site and department (ROUTE_SITE_NAME and ROUTE_DEPT_NAME). Routing to a specific user (ROUTE_USER_NAME) in the destination site and department is optional.

      

Column

Required Field

Description

SUPPLIER_NAME

 

Enter the supplier company name for the rule to consider invoices when the supplier company name matches the invoice’s supplier company name.

Note: An asterisk (*) may follow letters/words to include all supplier names beginning with those letters/words.

SUPPLIER_SITE_NAME

 

Enter the supplier site name for the rule to consider invoices when the supplier site name matches the invoice’s supplier site name.

REMIT_SITE_NAME

 

Enter the Remit site name for the rule to consider invoices when the Remit site name matches the invoice’s Remit site name.

REMIT_DEPT_NAME

 

Enter the Remit department name for the rule to consider invoices when the Remit department name matches the invoice’s Remit department name.

ROUTING_CODE

 

Enter the value used to specify the invoice destination (max 100 characters)

ROUTE_SITE_NAME

 

Enter the invoice site destination

ROUTE_DEPT_NAME

 

Enter the invoice department destination

ROUTE_USER_NAME

 

Enter the invoice recipient

ROUTE_REQUISITIONER

 

Enter TRUE to route the invoice to the first requisitioner listed on the invoice. This value is case-sensitive.

Note: If you enter TRUE in this field all other fields must be blank.
