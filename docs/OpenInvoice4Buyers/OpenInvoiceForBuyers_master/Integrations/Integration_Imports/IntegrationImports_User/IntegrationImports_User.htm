User Import

The user information files used in your company’s back office is imported to ensure the OpenInvoice Platform corresponds. Information on each user such as name, contact information, approval levels, authorization limits, and delegations, can be included in the file. Each User Import file may include multiple users and associated business rules.

User Import files are in XML format. The maximum size of each file is 3 MB. Larger files may time out and result in a failed import.

The User Import is not a scheduled service. Instead, you will post the files as necessary or you can schedule the service in OpenInvoice. The file is not saved on the secure server but is, instead, redirected to the OpenInvoice Platform if not posted there directly.

Business Rules for Usernames

The UserName element is the user’s sign in name. If this element is not present, the base username is derived from the user’s first name, middle initial, and last name.

Usernames are never updated for existing users. If the username already exists when creating a new user, a suffix will be added to the base username with the lowest possible integer in order to get a unique username. For example, if the import identifies a new user with <Username>joehsmith</Username> or <FirstName>Joe</FirstName> <Initial>H</Initial> <LastName>Smith</LastName> and joehsmith, joehsmith1, joehsmith2 and joehsmith123 already exist as system usernames, the assigned username for the new user will be joehsmith3.

If a username is less than six characters, it will be padded with zeros and the lowest possible integer suffix to get a unique username. For example, <Username>hge</Username> would result in the username hge001.

User profile email addresses support the use of ampersand (&), apostrophe (') and plus (+) characters. The ampersand and apostrophe characters must be encoded in the import file as &amp; and &apos; respectively. For example, Joe.O’Mally@nobody.com is encoded as Joe.O&apos;Mally@nobody.com.

Each time a new user is created during an import, a warning message will be added to the response file and sent back to the buyer.
