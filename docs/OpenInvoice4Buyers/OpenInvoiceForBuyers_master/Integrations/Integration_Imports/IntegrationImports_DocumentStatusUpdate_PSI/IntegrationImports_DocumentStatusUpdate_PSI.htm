Document Status Update Import (PSI)

The Document Status Update Import allows buyers to update invoice statuses based on the actions performed in their back office. Multiple invoices can be updated in each Document Status Update file. The file contains the invoice identifier for each invoice and the action that has been taken on the invoice.

Document Status Updates are transmitted in multi-part MIME format.

The invoice can be marked with the following actions. With the exception of the comment action, an action can only be taken once on an invoice. If multiple actions are posted for an invoice, the actions will be applied to the invoice ordered by the TransactionDatetime value.

·Post

·Pay

·Post Cancel

·Pay Cancel

·Comment

·ReleaseForPayment

These attributes associated with the Pay and Post actions can also be included.

·Whether the buyer has realized an early pay discount.

·The total amount the buyer has paid on an invoice. This tag does not need to be specified if the buyer does not use the OpenInvoice interface. The invoice will be treated as short paid if the remit total is less than invoice amount.

Schema: DocumentStatusUpdateBatch.xsd
