Purchase Order and PO Change Request

Purchase orders and purchase order change requests are imported into the OpenInvoice Platform. The PIDX version 1.2 OrderCreate schema is used for sending the initial purchase order, and PIDX version 1.2 OrderChange is used for change requests. Once the purchase order has been created in the OpenInvoice Platform, subsequent changes made to the purchase order must be sent by Order Change. Order Change must be the complete purchase order. Any data not included in the Order Change will be assumed to have been deleted from the purchase order.

All Order Changes posted by a client must be sequenced to ensure that the net result of two or more changes are applied properly.

The message is SOAP wrapped if it does not include attachments, or in multipart MIME format if it does.

Schemas: PIDX version 1.2 OrderCreate.xsd and OrderChange.xsd

See also: PurchaseOrderFromBuyer_IG.pdf

Auto-Accepted Purchase Orders

Purchase order responses are not sent from integrated suppliers. Buyers can post purchase orders and change requests to the system with a status of Accepted by giving the SpecialInstructions tag in the PIDX schema a value of Accepted as follows:

<pidx:SpecialInstructions instructionIndicator=”Other” definitionOfOther=”DocumentState”>Accepted</pidx:SpecialInstructions>

If the flag is not turned on, the supplier must log in to the system and manually accept or reject the purchase order or change request.

Maximum Number of Purchase Order Lines

For most suppliers, purchase orders should be kept to 500 lines or less due to potential performance issues during the flip process. The greater the number of products and services that a supplier has, the lower the number of lines that can be flipped before a system time out occurs. Suppliers with very large catalogs may only be able to flip purchase orders of 200 lines or less.
