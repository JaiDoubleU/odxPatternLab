Unapproved Invoice Batch Export

The Unapproved Invoice Batch Export allows XML data for multiple invoices to be bundled in one XML file and delivered to the buyer. Exporting multiple invoices with one server connection minimizes the impact on overall system performance. The data can be used for internal reporting/cost estimation or for invoice adjudication purposes.

The buyer must be able to extract individual invoice data out of an invoice batch file.

Attachments are not included.

Messages are SOAP wrapped.

Both image and detail invoices can be exported. Tags in the XML are used to differentiate between image and detail invoices. You can enable an option to trigger the Unapproved Invoice Batch Export when an image invoice is deleted in OpenInvoice. Contact OpenInvoice Support to enable this option.

Includes original submissions and resubmissions.

Scheduled Service: Unapproved Invoice Batch Export

Schema: SupplierConnectInvoiceBatch.xsd

See also: Unapproved Invoice Batch Export.pdf

To manually re-export invoices that were previously exported, click the action icon, then Select Invoices to search for invoices you wish to export. On the Invoice Search Results page, select the invoices and click Submit to export the invoices immediately.   

·Schedule: run at specified time(s) every day.

·Triggers: Any of the following actions can add an invoice to the pending export list so it is exported the next time the service is run. If none of the selected actions were performed on the invoice since the service was last run, it is not included in the next service.

§Submit: invoices are added to the next export service when they are submitted or resubmitted by the supplier. There is an option to have any invoices automatically disputed by OpenInvoice on your behalf excluded from the export.

The rest of the options can be set as triggers only if the invoice is processed in the OpenInvoice Platform. The export will update the back office with the invoice’s processing stage.

·Filemask: The location where the file will be sent and name of the created file.

·Use legacy OI-XML file format for exported invoices: This setting is for companies using an older OpenInvoice invoice XML schema.

·Use Legacy Entry Point: This defines whether the invoices use the PIDX Invoice (version 1.2) schema or OpenInvoice’s SupplierConnect schema.

·Export File Size: OpenInvoice does not set file size limits (except on attachments). However, the system may time out on files larger than 3 MB, so companies may want to set a file size limit to reduce the number of failed transfers. If the invoice batch file size exceeds the size specified here, the invoices will be split into separate batches (i.e., this setting indicates the size at which the batch will be split into separate batches). For example, if 200 is entered as the export file size and the submitted batch is over this amount, the batch will be split into separate batches of up to 200 KB. An invoice will not be split across batches.
