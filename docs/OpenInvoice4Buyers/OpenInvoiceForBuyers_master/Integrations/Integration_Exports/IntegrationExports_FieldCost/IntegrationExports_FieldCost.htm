Field Cost Export

All modifications made in the ticket module since the last time the export ran will be exported to the your company’s back office. This includes job sites and activities being added or closed, and charges being added, cancelled, or approved. This allows buyers to be aware of the current expected costs. This service is an incremental export. If nothing has changed in the ticket module, the file is not exported.

Messages are transmitted in XML format.

·Schedule: can be run at specified time(s) every day.

Export File Size: OpenInvoice does not set file size limits (except on attachments). However, the system may time out on files larger than 3 MB, so companies may want to set a file size limit to reduce the number of failed transfers.

Scheduled Service: OI Ticket Charge Export service settings. If nothing has changed in the ticket module, the file is not exported.

Schema: FieldCost.xsd

See also: The Field Cost Export does not have an implementation guide.
