Payment Status

All modifications made in the OpenInvoice payment module since the last time the service ran will be exported to the your company’s back office. The information is exported in a CSV file which includes a header row with the field (column) names and one row for each payment included in the file. Each field (column) in the file is separated by a comma. If the value within a field contains a comma, the value will be enclosed in quotes.

Payment Status Exports provide the current status of multiple payments in a single transaction. Only payments modified since the last time the export ran will be exported. This service is an incremental export. If nothing has changed in the OpenInvoice payment module, the file is not exported. Users who manually run the service will be notified in the Message Center if the service failed or if there were no records found to export.

Messages are transmitted in CSV format.

·Schedule: can be run at specified time(s) every day.

·Filemask: The location where the file will be sent and name of the created file.
