Approved Invoices Export

Invoices approved in the OpenInvoice Platform are exported to the secure server and/or your company’s back office. Approved invoices are exported in two scenarios:

·invoices are coded and final approved within the OpenInvoice Platform

·invoices are approved in the buyer’s back office and an Invoice Response file is sent to mark the invoice as approved in the OpenInvoice Platform.

The export transfers the final approved invoice to the buyer’s back office.

Approved Invoice Exports are transmitted in multi-part MIME format.

Both image and detail invoices can be exported. Different tag names in the XML are used to differentiate between image and detail invoices.

Click the action icon, then Select Invoices to search for invoices you wish to export. Otherwise, all approved invoices will be exported according to the service configuration setting.

·Schedule: run at specified time(s) of the day or in real time (invoices are exported within a few minutes of being approved in the OpenInvoice Platform regardless of the approval method— manual, automatic or integrated).

·Include All Attachments: If set to Yes, any attached backup documentation such as scanned invoices will be exported as well.

·Generate PDF of Invoice on Export: If set to Yes, a PDF of the invoice will be created and transferred as an attachment along with the XML file.

·Attachment/PDF Directory: Location for any attachments or generated PDFs.

·Filemask: The location where the file will be sent and name of the created file.

Scheduled Service: Approved Invoice Export

Schema: OpenInvoice’s SupplierConnectInvoice.xsd
