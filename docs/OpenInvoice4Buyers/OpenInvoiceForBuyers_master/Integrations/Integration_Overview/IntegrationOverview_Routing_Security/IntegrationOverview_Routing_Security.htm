Routing and Security

All communications between the OpenInvoice Platform and client servers will be enabled through HTTP version 1.1 over SSL. Conversations will be secured using two-way SSL and will only be established with servers supporting a strong encryption algorithm with a minimum key length of 128 bits.

Non-Repudiation

Non-repudiation ensures that a transferred message has been sent and received. A comprehensive non-repudiation plan includes authentication, authorization, data integrity, and auditing.

Authentication

Authentication verifies the identity of the sender of a message. The OpenInvoice Platform uses two-way certificate-based authentication, which requires the sender to present their digital certificate to negotiate the conversation. The certificate presented is then verified against a copy of the client public certificate residing in the OpenInvoice Platform.

Authorization

Authorization ensures that the authenticated client is permitted to perform the service they are requesting. The OpenInvoice Platform allows access to various resources based on the client’s identity. Entry points to the OpenInvoice Platform are secured through the use of Access Control Lists (ACLs). The client’s identity is assigned to one or more ACLs.

Data Integrity

The OpenInvoice Platform achieves data integrity by not altering the document XML that is contained in the SOAP body of the transmitted message. It will parse the MIME message apart to obtain the attachments and save them on the file system, but will not alter the document XML.
