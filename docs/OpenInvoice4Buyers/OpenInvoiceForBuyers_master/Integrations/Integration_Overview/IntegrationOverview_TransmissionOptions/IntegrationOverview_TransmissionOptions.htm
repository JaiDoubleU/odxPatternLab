Transmission Options

Files can be posted directly to and from a URL in the OpenInvoice Platform and a URL in your company’s back office. Files can also be transferred between your back office and the OpenInvoice Platform via an Oildex-maintained secure server residing on your network.

As part of buyer company implementation, a transmission method is decided upon and OpenInvoice Support and the company’s technical team will work together to configure secure communication between the OpenInvoice Platform, the company’s back office and, if applicable, the secure server. Once a method is established, the company will use the same method for all of their file transfers.

Using a Secure Server

Buyer companies can also use an Oildex-maintained secure server to support the secure transmission of data to and from the company’s back-office system (redirect method). The secure server can also serve as a temporary file repository. Files exported from the OpenInvoice Platform are saved to a predefined directory on the secure server for the client to retrieve for processing (save method).

The secure server is software that is installed on the buyer company’s network in the DMZ between the buyer company’s network and the OpenInvoice Platform. This provides an additional level of security since messages are not being transferred in and out of the company’s network directly.

OpenInvoice Support provides the Certificate Authority services required for the server-to-server communications. A Certificate Management Server (CMS) is used to perform Certificate Authority administrative functions such as generation of certificates, expiry/renewal, and the revocation of certificates. It issues and manages X.509 certificates. Certificates issued by all major Certificate Authorities, such as VeriSign, are also fully supported.

Imports

Files being transferred from the buyer’s back office to the OpenInvoice Platform are posted to the secure server, which then establishes the SSL conversation with the OpenInvoice Platform for the transfer of documents. The file system and the location in the OpenInvoice Platform are set up during the initial integration setup. The connection between the secure server and the buyer company back office URL is enabled through HTTP protocol with username/password authentication. The username and password will be provided by OpenInvoice Support.

Some of the buyer imports that do not have associated scheduled services (ADE Import, Cost Collection Import, Document Status Update, Purchase Order Import, and User Import) are not saved on the secure server. The files must be redirected from the secure server or posted directly to the OpenInvoice Platform.

Exports

Two options are available for companies using a secure server to receive files.

Save Option

Files are saved in a configured folder on the secure server. The file system and the location in the buyer’s back office are set up by OpenInvoice Support as part of the electronic document configuration. Approved and unapproved invoices will also each have a folder for attachments and generated invoice PDFs.

·The MIME and SOAP wrappers are removed from the file. The buyer will retrieve the XML from the secure server. The encoded attachments are taken out, restored to their original format, and put in the designated attachment folder.

·A semaphore file will be written to the same directory as the XML file when the file transfer is complete. The semaphore file is given the same name as the XML file with the addition of a .done extension to indicate to the client that the XML and attachment files are ready to be retrieved.

·Folder maintenance and archiving of data are the responsibility of the client. OpenInvoice does not remove any files from the directory.

·The buyer company retrieves the XML files and the attachments (if any) from the secure server, extracts the XML data, and organizes it into the format required by their back office system.

Redirect Option

Files are streamed to the secure server which then routes the data stream to a buyer-specified server URL. The connection between the secure server and the buyer company back office URL is enabled through HTTP protocol with username/password authentication. The username and password are provided by the buyer company. Invoice attachments are unzipped and saved on the secure server.

The MIME and SOAP wrappers are not removed from the XML message in the redirect method.

Without a Secure Server

Documents can be transmitted directly to and from the OpenInvoice Platform using the company’s current infrastructure.

Exports are posted directly to a company-specified URL. Imports are posted from the company’s back office to a URL in the OpenInvoice Platform. Communication is via HTTPS/SSL protocol with two-way certificate exchange.

To successfully configure the connection between a company’s back end server and the OpenInvoice Platform, the following tasks must be completed. OpenInvoice Support will provide a template file to the company for completion.

·Exchange SSL certificates: The OpenInvoice Platform is configured to perform two-way authentication. The company must install the appropriate (test or production) OpenInvoice public key and issuing CA (Certificate Authority) to the trusted store of their back office server. The company must also provide OpenInvoice Support with copies of the public key and issuing CA that will be installed to the OpenInvoice Platform. OpenInvoice does not support the use of self-signed certificates.

·Exchange IP address details: OpenInvoice’s test and production environments are fully independent of each other. Each environment features a unique IP address and URL. OpenInvoice’s security policy allows companies to specify one IP address to be configured for each environment (test and production).

·Exchange DUNS numbers: A unique nine-digit DUNS number is required to identify the trading partners involved in transactions. DUNS numbers can be obtained from Dun & Bradstreet. Refer to the website www.dnb.com for more information.
