Integration Overview

Integration refers to the various methods of communication between your company’s internal financial system and the OpenInvoice Platform. OpenInvoice is the mediator or hub for integration; the process does not require direct integration between buyer and supplier. This allows each company to use a single method of transferring data to communicate with multiple trading partners who may have several different back office programs and methods of interacting with the OpenInvoice Platform.

Messages transmitted from the OpenInvoice Platform are kept in their original XML format before they are posted to the client. The document in its original form is retained for a period of time. Audit logs are also maintained. Attachments are not kept in the Trading Networks repository. If using a secure server, the location on the server and the naming convention for the particular file type are configured in the Integration Services function. If not using a secure server, the URLs for file transfers are set up by OpenInvoice Support in a different location and are not listed on the Integration Services page. The file will be sent to the location established during the initial setup.

Naming conventions are also determined as part of the initial configuration process. The file mask consists of a file name plus variables that create a time stamp to ensure each file name is unique. For example, files transmitted from the OpenInvoice Platform to a company's back office in real time may have the month, day, hour, minute, and second stamp to ensure that files arriving in rapid succession have unique names. Buyer configuration file imports (AFE, Cost Center, ADE, Cost Collection, and User Imports) will have the file name plus the month and day (AFE_0830 for an AFE file posted on August 30). The OpenInvoice Platform will retrieve a file with the agreed-upon prefix/suffix and the current date. Files that do not follow the agreed-upon naming convention are not recognized and, therefore, will not be transmitted.

This section provides an overview of the integration process and the types of information that can be transferred between OpenInvoice and your company. During the integration process, OpenInvoice Support will provide detailed documentation describing the different integration types and the business rules associated with each XML schema.
