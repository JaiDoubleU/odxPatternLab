Code and Approve Field Tickets

Buyers can edit select fields on a submitted field ticket, and the fields available for editing depend on the content model implemented by your company. Buyers can edit price books, as well as include a Superintendent and Office Approver, if appropriate.

How to Edit a Submitted Field Ticket

Permission:Field Ticket Modify or Field Ticket Modify (restricted)

Note:The Field Ticket Modify (restricted) permission only allows you to modify tickets owned by you in a department where this permission is assigned. You own a ticket in a department either directly (when you are specified in the “owner” field) or indirectly (when the department name is specified in the “owner” field).

Steps:

  1. Click Ticket > Ticket Application.

  2. Enter Field Ticket Number in the Search field, then click the Search icon.

  3. Click the ticket.

    4.  

  4. Click the Edit Ticket Header button.

    5.  

  5. Update the following fields:

    6. §Price Book Name: Select a price book from the drop-down list. The drop-down list will automatically populate with all active, non-terminated price books (expired or not), valid for at least one of the service dates on the ticket plus a two week prior window to ensure availability of recently expired price books for new lines being created. Expired price books are listed last.

    §Price Book Description: Price book description is auto-populated based on the price book name selected.

    §Superintendent: Enter the name of the person from the buying company who ordered the good or service, or use the Search function to locate the appropriate user.

    §Office Approver: Enter the name of the person from the buying company who is responsible for approving the ticket, or use the Search function to locate the appropriate user.

  6. Click Save to update the ticket header information.

Field Ticket Coding

Buyers configure the information that is required to be submitted on a ticket by the supplier. If configured by the buyer, suppliers may include AFE, Cost Center, Major/Minor/Subcode and PO information on submitted tickets. Buyers have the option to include AFE, Cost Center, Major/Minor/Subcode and PO information on tickets if the information is not submitted by the supplier and may edit submitted information in these fields, if necessary.

Coding information from the ticket will be transferred from the ticket to a submitted invoice that references the coded ticket.

Tickets can contain Common Coding or Detailed Coding. Common coding has the AFE, Cost Center, Major/Minor/Subcode and PO data at the header level and all line items use the same coding. Detailed coding allows for each line item to have unique AFE, Cost Center, Major/Minor/Subcode and PO assignments.

AFE, Cost Center, and Major/Minor/Subcodes must be valid codes. Users can search for AFEs and Cost Centers.

Common Coding

Common coding appears at the header level where AFE, Cost Center, Major/Minor/Subcode and PO data applies to all lines on a ticket. To apply common coding, see Add Common Coding to a Field Ticket.

 

AFE and Cost Center codes can be entered manually or users can click on the Search icon to search for a code. Major/Minor/Subcode and PO can be entered manually. If there is only one minor code for the entered major code the minor code will auto-populate. The subcode will auto-populate if there is only one subcode for the minor code.

Note:Coding changes are automatically saved to the ticket when the user navigates away from the field.

Detailed Coding

Detailed coding appears on each line item of a ticket and can be unique for each line item. If common coding already exists on an invoice the codes will be moved to the line items and can be edited. Select Detailed Coding to add coding to all line items on a ticket.

Note:Coding changes are automatically saved to the ticket when the user navigates away from the field.

How to Add Common Coding to a Field Ticket

Ticket approvers have the ability to apply coding to a field ticket.

Steps:

  1. Click Ticket > Ticket Application.

  2. Enter Field Ticket Number in the Search field, then click the Search icon.

  3. Click the ticket.

    4.  

  4. Click the Edit Ticket Header button.

  5. Enter the AFE and Cost Center.

    6.  

  6. Click Save.

How to Allocate a Field Ticket to Multiple AFEs and/or Cost Centers

Ticket approvers have the ability to allocate a field ticket to multiple AFEs and/or/ Cost Centers.

Steps:

  1. Click Ticket > Ticket Application.

  2. Enter Field Ticket Number in the Search field, then click the Search icon.

  3. Click the ticket.

  4. Click the Edit Ticket Header button.

    5.  

  5. Click the Add AFE/Cost Center button. A new row appears.

    6.  

  6. Type the AFE and/or Cost Center.

    7.  

    Note:The allocation percentage and amount will be auto-split based on the number of AFE/Cost Centers added to the field ticket.

  7. Click Save to save the new AFE and/or Cost Center allocation.

How to Approve a Submitted Field Ticket

Buyers can approve submitted tickets from suppliers.

Permission:Field Ticket Modify or Field Ticket Modify (restricted)

Note:The Field Ticket Modify (restricted) permission only allows you to modify tickets owned by you in a department where this permission is assigned. You own a ticket in a department either directly (when you are specified in the “owner” field) or indirectly (when the department name is specified in the “owner” field).

Steps:

  1. Click Ticket > Ticket Application.

  2. Enter Field Ticket Number in the Search field, then click the Search icon.

  3. Click the ticket.

  4. Click the Approve button. The Approve Ticket window displays.

    5.  

  5. Click Yes, approve ticket to complete the approval.

Note:Coding changes are automatically saved to the ticket when the user navigates away from the ticket.
