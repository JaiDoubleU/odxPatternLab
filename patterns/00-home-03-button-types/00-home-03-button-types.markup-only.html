Button Usage

Supporting Principles
  • Use default buttons in all circumstances, unless otherwise specified.
  • Keep button text to as few words as possible.
  • Follow the Verb - Noun format (i.e. "Submit Form", "Generate Report", "Submit Invoice", etc.)
Button Types: Why so many colors?

Contextual buttons provide colors to communicate a contextual call to action that performs an action or provides navigation. These contextual colors are typically used to distinguish a button from the default button style.

Default Buttons
The standard button to use in all circumstances, unless otherwise specified.

CSS Class: btn-default

Primary Buttons
Provides extra visual weight and identifies the primary action on a page

CSS Class: btn-primary

Success Buttons
Indicates a positive action taken or an action that advances the user to a next step or phase

CSS Class: btn-success

Info Buttons
A contextual button for use in informational alert messages or modal dialogs
CSS Class: btn-info
Warning Buttons
A contextual button for use in warning alert messages or modal dialogs.

CSS Class: btn-warning

Danger Buttons
A contextual button that indicates a dangerous or potentially negative action, for use in error alert messages or modal dialogs

CSS Class: btn-danger

Link Buttons
Makes buttons appear and behave as links, while maintaining the minimum height and width for a clickable element

CSS Class: btn-link

Button Groups: To group or not to group?

Use button groups to combine a series of related functions together on a single line simulating the appearance of a toolbar.
